1 — Rappels : la programmation orientée objet

Imagine que tu veuilles représenter un point dans l'espace 3D. Tu pourrais utiliser trois variables séparées : x, y, z. Mais si tu dois gérer des centaines de points, ça devient vite ingérable. La programmation orientée objet (POO) résout ce problème en regroupant des données et des comportements dans une seule entité : une classe.

Définitions essentielles
TermeDéfinition
ClasseModèle (gabarit) qui décrit la structure et les comportements d'un type d'objet.
InstanceUn objet concret créé à partir d'une classe. Chaque instance est indépendante.
AttributVariable appartenant à une instance (stockée dans self).
MéthodeFonction définie dans une classe, qui agit sur l'instance via self.
__init__Constructeur — appelé automatiquement lors de la création d'une instance.
selfRéférence à l'instance courante. Toujours premier paramètre d'une méthode.
Exemple minimal – la classe Point2D 
class Point2D:
    def __init__(self, x, y):
        self.x = x       # attribut x
        self.y = y       # attribut y

    def decrire(self):
        return f"Point ({self.x}, {self.y})"

p = Point2D(3, 4)         # création d'une instance
print(p.decrire())        # → "Point (3, 4)"
print(p.x)                # → 3  (accès à un attribut)

Appel de méthode : deux notations équivalentes

Quand tu écris p.decrire(), Python traduit cela en Point2D.decrire(p). Le premier paramètre (self) reçoit automatiquement l'instance p. Tu n'as jamais à passer self explicitement lors de l'appel.

Appel de méthode d'une autre instance

Une méthode peut recevoir une autre instance en paramètre. C'est exactement ce qui se passe dans ce sujet : s1.est_adjacent(s2) teste si le sommet s2 est adjacent à s1. À l'intérieur de la méthode, self désigne s1 et le paramètre sommet désigne s2.

Le format OBJ – comment lire un fichier

Structure d'un fichier OBJ 
o cube            ← nom de l'objet
v 0.0 0.0 0.0    ← sommet 1 : (x=0, y=0, z=0)
v 0.0 1.0 0.0    ← sommet 2 : (x=0, y=1, z=0)
v 1.0 1.0 0.0    ← sommet 3 : (x=1, y=1, z=0)
v 1.0 0.0 0.0    ← sommet 4 : (x=1, y=0, z=0)
f 1 2 3 4        ← face reliant les sommets 1, 2, 3, 4 (1-indexé)

Attention : les indices dans les lignes f commencent à 1 (convention OBJ), alors que les listes Python commencent à 0. C'est pourquoi afficher() utilise self.sommets[s-1].

La méthode est_adjacent – déjà fournie

Deux sommets sont adjacents (reliés par une arête) si et seulement si leurs coordonnées ne diffèrent que sur un seul axe. Par exemple, (0,0,0) et (1,0,0) ne diffèrent que sur x → adjacents. Mais (0,0,0) et (1,1,0) diffèrent sur x et y → non adjacents. 

def est_adjacent(self, sommet):
    nb_changement = 0
    if self.x != sommet.x: nb_changement += 1
    if self.y != sommet.y: nb_changement += 1
    if self.z != sommet.z: nb_changement += 1
    return nb_changement == 1

Question 1 — Méthode distance dans Sommet

Énoncé — Question 1

Q1 Écrire la méthode distance de la classe Sommet. Elle prend en paramètre un objet de type Sommet et renvoie la distance entre l'objet courant et ce sommet.

Rappel de la formule — distance entre \(A(x_A, y_A, z_A)\) et \(B(x_B, y_B, z_B)\) : \[d(A, B) = \sqrt{(x_A - x_B)^2 + (y_A - y_B)^2 + (z_A - z_B)^2}\] Utiliser math.sqrt pour calculer la racine carrée.

Aide méthodologique

Démarche

Question 2 — Méthode trouver_sommets_adjacents

Énoncé — Question 2

Q2 Écrire la méthode trouver_sommets_adjacents de la classe Objet3D. Elle renvoie un tuple composé de deux sommets adjacents, ou None si aucun couple n'est trouvé.

La méthode doit parcourir tous les couples possibles de sommets avec deux boucles imbriquées et utiliser la méthode est_adjacent. Elle renvoie le premier couple trouvé.

Aide méthodologique

Démarche — Parcourir tous les couples sans doublon

Pour parcourir tous les couples (i, j) avec i < j (afin d'éviter de tester deux fois le même couple et d'éviter (i, i)) : 

for i in range(len(self.sommets)):
    for j in range(i + 1, len(self.sommets)):
        # tester self.sommets[i] et self.sommets[j]

Dès qu'on trouve un couple adjacent, on le renvoie immédiatement avec return. Si la double boucle se termine sans rien trouver, on renvoie None.

Indice 1

Pour tester si les sommets i et j sont adjacents : self.sommets[i].est_adjacent(self.sommets[j]). Si c'est vrai, renvoie le tuple (self.sommets[i], self.sommets[j]).

Indice 2 
def trouver_sommets_adjacents(self):
    for i in range(len(self.sommets)):
        for j in range(i + 1, len(self.sommets)):
            if self.sommets[i].est_adjacent(self.sommets[j]):
                return (self.sommets[i], self.sommets[j])
    return None
Éditeur Python — Question 2
Correction — Question 2

La boucle interne part de j = i + 1 (et non j = 0) pour deux raisons : éviter de tester (i, i) (un sommet avec lui-même) et éviter les doublons (i, j) et (j, i).

Le return None final est techniquement inutile en Python (une fonction sans return renvoie déjà None), mais il est important de l'écrire explicitement pour la lisibilité et pour montrer qu'on a bien traité ce cas.

Question 3 — Fonction estimation_impression

Énoncé — Question 3

Q3 Écrire la fonction estimation_impression(volume, parametres) qui prend en paramètre le volume réel d'un objet (entier) et un dictionnaire de paramètres d'imprimante. Elle renvoie le temps total d'impression en secondes (flottant).

  1. Calculer le volume d'impression = volume réel × taux de remplissage (clé 'remplissage').
  2. Calculer le temps = volume d'impression ÷ vitesse d'extrusion (clé 'vitesse_extrusion').

Aide méthodologique

Contexte – les paramètres de l'imprimante 
parametres_imprimante = {
    'remplissage': 20,           # taux en pourcentage : 20 %
    'vitesse_extrusion': 8       # vitesse en mm³/s
}

Le taux de remplissage est en pourcentage — il faut donc diviser par 100 pour obtenir un coefficient multiplicateur. Un taux de 20 % signifie que l'imprimante dépose réellement 20 % du volume du cube.

Exemple chiffré

Cube de côté 10 mm → volume = 1000 mm³.

  • Volume d'impression = 1000 × 20 / 100 = 200 mm³
  • Temps = 200 ÷ 8 = 25 secondes
Indice

Pour accéder aux valeurs du dictionnaire : parametres['remplissage'] et parametres['vitesse_extrusion']. Attention à l'orthographe exacte des clés — Python est sensible à la casse.

Éditeur Python — Question 3
Correction — Question 3

La division par 100 est indispensable pour convertir le pourcentage en coefficient. Sans elle, le volume d'impression serait 100 fois trop grand. La formule complète en une ligne est aussi valide : return volume * parametres['remplissage'] / 100 / parametres['vitesse_extrusion'].

Question 4 — Construire et afficher une pyramide

Énoncé — Question 4

Q4 Modifier le programme de main.py pour instancier et afficher la pyramide dont les sommets sont : (0,0,0), (2,0,0), (0,2,0), (2,2,0) (base carrée) et (1,1,2) (apex).

Aide méthodologique

Anatomie d'une pyramide à base carrée

Une pyramide à base carrée possède :

  • 5 sommets : 4 sommets de la base + 1 apex (sommet du haut)
  • 5 faces : 1 face carrée (la base) + 4 faces triangulaires (les côtés)
Correspondance sommets ↔ indices OBJ
Indice OBJCoordonnéesRôle
1(0, 0, 0)Base — coin avant gauche
2(2, 0, 0)Base — coin avant droit
3(0, 2, 0)Base — coin arrière gauche
4(2, 2, 0)Base — coin arrière droit
5(1, 1, 2)Apex (sommet de la pyramide)
Indice 1

La base relie les 4 coins : [1, 2, 4, 3] (dans l'ordre pour que la face soit plane).
Les 4 faces triangulaires sont : [1, 2, 5], [2, 4, 5], [4, 3, 5], [3, 1, 5].

L'ordre des sommets dans chaque face n'est pas anodin : pour que la face soit visible du bon côté (normale sortante), il faut respecter un sens de parcours cohérent. Matplotlib est assez permissif à ce sujet, mais dans des moteurs 3D réels (back-face culling), l'ordre compte beaucoup.

Question 5 — Corriger et améliorer transformer

Énoncé — Question 5

Q5 Analyser la méthode transformer fournie. Identifier son bug. Proposer une version corrigée qui, au lieu de modifier l'instance courante, renvoie un nouvel objet résultant de la transformation. Tester sur la pyramide de Q4 avec un rapport de 2.

Analyse du bug — code fourni

def transformer(self, rapport):
    sommets = []
    for sommet in self.sommets:
        sommets.append(
            Sommet(sommet.x*rapport, sommet.y*rapport, sommet.z*rapport))
    self.sommet = sommets   # ← BUG ICI
Identification du bug

La ligne self.sommet = sommets crée un nouvel attribut sommet (sans s), au lieu de modifier l'attribut existant self.sommets (avec s). C'est une faute de frappe — une seule lettre manquante — mais le résultat est que l'objet n'est jamais réellement modifié. L'attribut self.sommets reste inchangé, et un attribut fantôme self.sommet est créé inutilement.

Pourquoi renvoyer un nouvel objet ?

Le sujet demande d'aller plus loin : plutôt que de modifier l'objet en place (mutation), la méthode doit créer et renvoyer un nouvel Objet3D avec les sommets transformés. L'objet d'origine reste intact — on peut ainsi appliquer plusieurs transformations indépendantes sans perdre l'original.

Démarche
  1. Créer un nouvel Objet3D vide.
  2. Pour chaque sommet de self, ajouter un sommet multiplié par rapport au nouvel objet.
  3. Copier les faces de self dans le nouvel objet (les faces ne changent pas — seuls les sommets bougent).
  4. Renvoyer le nouvel objet avec return.
Indice 
def transformer(self, rapport):
    nouvel_objet = Objet3D()
    for sommet in self.sommets:
        nouvel_objet.ajouter_sommet(
            sommet.x * rapport,
            sommet.y * rapport,
            sommet.z * rapport)
    for face in self.faces:
        nouvel_objet.ajouter_face(face.sommets)
    return nouvel_objet
Éditeur Python — Question 5
Correction — Question 5

Le point clé : on ne fait jamais self.sommets = ... dans la méthode corrigée. L'instance d'origine reste intacte. Ce patron de conception (créer un nouvel objet plutôt que muter l'existant) est appelé objet immuable en POO et facilite grandement le débogage.

Note sur les faces : on copie la liste face.sommets telle quelle dans le nouvel objet. Les indices de sommets restent corrects car les sommets sont dans le même ordre dans les deux objets.

Pièges fréquents & astuces de vérification

Piège 1 — Confondre self.sommet et self.sommets

C'est le bug exact de la Q5. self.sommet (sans s) et self.sommets (avec s) sont deux attributs distincts. Python ne lève pas d'erreur quand tu crées un nouvel attribut par inadvertance — l'objet continue à fonctionner, mais avec les anciennes valeurs. Vérifie toujours l'orthographe de tes attributs avec print(vars(objet)).

Piège 2 — Indices OBJ commencent à 1, Python à 0

Dans un fichier OBJ, les sommets sont numérotés à partir de 1. Dans la liste Python self.sommets, ils sont indexés à partir de 0. C'est pourquoi afficher() utilise self.sommets[s-1]. Si tu oublies le -1, tu décales tous les sommets d'un cran et tu obtiens soit un mauvais rendu, soit une IndexError sur le dernier sommet.

Piège 3 — Boucle imbriquée : partir de i+1, pas de 0

Dans trouver_sommets_adjacents, la boucle interne doit partir de j = i + 1 (et non j = 0). Si tu pars de 0, tu testes le couple (i, i) — un sommet n'est jamais adjacent à lui-même selon est_adjacent (0 changement), mais tu testes aussi les couples en double : (0, 1) puis (1, 0).

Piège 4 — Oublier de diviser le taux de remplissage par 100

Le dictionnaire stocke le taux en pourcentage (20), pas en coefficient (0.20). Sans la division par 100, le volume d'impression est 100 fois trop grand et le temps calculé est absurde. Vérifie toujours avec un cube de côté 10 mm : le temps doit être 25.0 secondes.

Piège 5 — Modifier l'objet au lieu d'en créer un nouveau

En Q5, si tu écris self.sommets = nouveaux_sommets, tu modifies l'objet d'origine et la méthode ne renvoie rien (None). L'appel grande = pyramide.transformer(2) attribuera None à grande. Toujours vérifier qu'une méthode censée renvoyer un objet contient bien un return.

Astuces de vérification
  • Vérifier les attributs d'un objet : print(vars(mon_objet)) affiche tous les attributs de l'instance sous forme de dictionnaire. Très utile pour détecter un attribut mal nommé.
  • Tester est_adjacent manuellement : Sommet(0,0,0).est_adjacent(Sommet(1,0,0)) → True, Sommet(0,0,0).est_adjacent(Sommet(1,1,0)) → False.
  • Triangle 3-4-5 pour distance : Sommet(0,0,0).distance(Sommet(3,4,0)) doit donner exactement 5.0.
  • Transformation de rapport 1 : objet.transformer(1) doit produire un objet dont tous les sommets ont les mêmes coordonnées que l'original.
  • Nombre de faces conservé : après transformation, len(nouvel_objet.faces) == len(original.faces) doit toujours être vrai.

Mini-exercices de vérification

Mini-exercice A — Calcul mental de distance

Sans Python, calcule la distance entre ces couples de sommets :

  1. Sommet(0,0,0) et Sommet(0,0,5)
  2. Sommet(1,1,1) et Sommet(4,5,1)
  3. Sommet(0,0,0) et Sommet(1,1,1)
  1. \(\sqrt{0^2+0^2+5^2} = \sqrt{25} = 5.0\)
  2. \(\sqrt{3^2+4^2+0^2} = \sqrt{9+16} = \sqrt{25} = 5.0\)
  3. \(\sqrt{1+1+1} = \sqrt{3} \approx 1.732\)
Mini-exercice B — Adjacence ou pas ?

Ces couples de sommets sont-ils adjacents selon est_adjacent ?

  1. (0,0,0) et (1,0,0)
  2. (0,0,0) et (1,1,0)
  3. (2,3,4) et (2,3,5)
  4. (0,0,0) et (0,0,0)
  1. ✅ Adjacents — seul x change (1 différence).
  2. ❌ Non adjacents — x et y changent (2 différences).
  3. ✅ Adjacents — seul z change (1 différence).
  4. ❌ Non adjacents — 0 différence (un sommet n'est pas adjacent à lui-même).
Récapitulatif — Ce que tu dois retenir
  • Un fichier OBJ liste des sommets (v) et des faces (f), avec des indices commençant à 1.
  • La méthode distance utilise la formule euclidienne 3D avec math.sqrt.
  • Pour parcourir tous les couples sans doublon : boucle interne de i+1 à la fin.
  • Le taux de remplissage est en pourcentage — diviser par 100 avant de multiplier par le volume.
  • Une pyramide à base carrée = 5 sommets + 5 faces (1 base + 4 faces triangulaires).
  • Le bug de transformer : self.sommet au lieu de self.sommets — une lettre qui change tout.
  • Pour créer un objet sans modifier l'original : instancier un nouvel Objet3D, le peupler, et le return.
