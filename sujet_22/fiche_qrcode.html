Contexte & structure du problème

Un QR code simplifié est une grille de cases noires (bit 1) et blanches (bit 0). Ici, chaque ligne de 8 cases forme un tuple de 8 bits, et le QR code entier est une liste de tuples.

Pipeline de décodage

QR code (liste de tuples binaires)
  → bin2dec sur chaque tuple → liste d'entiers décimaux
  → dec2str sur la liste → chaîne de caractères (message)

Rappel — Conversion binaire → décimal

Chaque bit a un poids qui est une puissance de 2, en partant de la droite (bit de poids faible). Pour un tuple de 8 bits (b₇, b₆, b₅, b₄, b₃, b₂, b₁, b₀) :

valeur = b₇×2⁷ + b₆×2⁶ + b₅×2⁵ + b₄×2⁴ + b₃×2³ + b₂×2² + b₁×2¹ + b₀×2⁰

Exemple : (0,1,1,0,0,0,0,1) → 0×128 + 1×64 + 1×32 + 0×16 + 0×8 + 0×4 + 0×2 + 1×1 = 97 → caractère 'a'

Question 1 — Fonction bin2dec

Question 1 / 4

Écrire une fonction bin2dec(tuple_bin) qui prend en paramètre un tuple représentant un nombre binaire et qui renvoie l'entier naturel en base 10 correspondant.

Puis, à l'aide de cette fonction, déterminer la chaîne de caractères contenue dans le QR code de la figure 1 pour découvrir le nom de l'inventeur de ce système de codage.

Analyse & méthode

Il faut parcourir le tuple de droite à gauche en multipliant chaque bit par la puissance de 2 correspondante, ou de gauche à droite en utilisant l'algorithme de Horner (plus élégant).

Deux approches possibles

Approche 1 — Par puissances de 2 : utiliser enumerate en inversant le tuple.

Approche 2 — Algorithme de Horner : accumuler bit par bit avec resultat = resultat * 2 + bit.

Indice

Pour l'approche par puissances : for i, bit in enumerate(reversed(tuple_bin)) puis resultat += bit * (2 ** i).

Pour Horner : initialiser resultat = 0, puis pour chaque bit : resultat = resultat * 2 + bit.

✔ Correction — deux approches

Approche 1 — Par puissances de 2 :

def bin2dec(tuple_bin):
    """Convertit un tuple de bits en entier décimal."""
    resultat = 0
    for i, bit in enumerate(reversed(tuple_bin)):
        resultat += bit * (2 ** i)
    return resultat

Approche 2 — Algorithme de Horner (recommandée) :

def bin2dec(tuple_bin):
    """Convertit un tuple de bits en entier décimal (méthode de Horner)."""
    resultat = 0
    for bit in tuple_bin:
        resultat = resultat * 2 + bit
    return resultat

Question 2 — Fonction qrcode2dec

Question 2 / 4

Écrire une fonction qrcode2dec(qrcode) qui prend en paramètre une liste de tuples représentant un QR code et qui renvoie une liste d'entiers décimaux correspondant à chacune des lignes.

Proposer un test qui utilisera la représentation du QR code de la figure 1 fournie dans ascii.py.

Indice

Cette fonction applique simplement bin2dec à chaque tuple de la liste. Une compréhension de liste est idéale : [bin2dec(t) for t in qrcode].

✔ Correction 
def qrcode2dec(qrcode):
    """Convertit une liste de tuples binaires en liste d'entiers décimaux."""
    return [bin2dec(ligne) for ligne in qrcode]

Question 3 — Correction de dec2str

Question 3 / 4

Exécuter la fonction fournie test_dec2str() et observer les résultats affichés. Identifier le problème et proposer une modification de dec2str pour l'éviter.

Après modification, dec2str devra toujours renvoyer une chaîne lisible.

Code fourni (bugué)

def dec2str(liste_dec):
    table_ascii = dict_ascii
    chaine = ""
    for entier in liste_dec:
        chaine += table_ascii[entier]   # ← BUG : KeyError si entier > 127
    return chaine

def test_dec2str():
    tests = [test1, test2, test3]
    for test in tests:
        print(dec2str(test))
🐛 Identification du bug

test3 contient l'entier 233 (le caractère é en Latin-1/ISO 8859-1), qui est hors de la table ASCII (0–127). L'accès table_ascii[233] lève une KeyError car 233 n'est pas une clé du dictionnaire dict_ascii.

Indice

Utilisez dict.get(clé, valeur_par_défaut) pour éviter le KeyError. Quelle valeur par défaut rendre si le code n'est pas dans la table ASCII ? Un caractère comme '?' ou '□' indique un caractère inconnu.

✔ Correction

Approche 1 — Avec .get() et valeur par défaut :

def dec2str(liste_dec):
    """Convertit une liste d'entiers en chaîne ASCII.
    Les codes hors table (>127) sont remplacés par '?'."""
    table_ascii = dict_ascii
    chaine = ""
    for entier in liste_dec:
        chaine += table_ascii.get(entier, "?")
    return chaine

Approche 2 — Avec in pour tester l'existence :

def dec2str(liste_dec):
    table_ascii = dict_ascii
    chaine = ""
    for entier in liste_dec:
        if entier in table_ascii:
            chaine += table_ascii[entier]
        else:
            chaine += "?"
    return chaine

Question 4 — Correction de str2qrcode

Question 4 / 4

La fonction str2qrcode(message) est censée effectuer l'opération inverse : convertir une chaîne de caractères en liste de tuples binaires (le QR code).

Cependant, en l'appliquant à la chaîne de la figure 1, on n'obtient pas exactement la figure 1. Analyser le code, identifier le bug, proposer une correction.

Code fourni (bugué)

def str2qrcode(message):
    qrcode = []
    table_inverse = {valeur: cle for cle, valeur in dict_ascii.items()}
    for caractere in message:
        entier = table_inverse.get(caractere, 63)
        binaire_str = bin(entier)[2:]          # ← BUG : longueur variable !
        ligne = tuple(int(bit) for bit in binaire_str)
        qrcode.append(ligne)
    return qrcode
🐛 Identification du bug

bin(entier)[2:] renvoie une chaîne binaire sans les zéros de tête. Par exemple :

  • bin(77)[2:] → '1001101' (7 bits) au lieu de '01001101' (8 bits)
  • bin(46)[2:] → '101110' (6 bits) au lieu de '00101110' (8 bits)

Les tuples générés n'ont donc pas tous 8 éléments, ce qui ne correspond pas au format du QR code (8 bits par ligne).

Indice

Il faut forcer la chaîne binaire à 8 caractères en ajoutant des zéros à gauche. Python offre deux syntaxes pratiques :
binaire_str.zfill(8) — remplit à gauche avec des '0' jusqu'à atteindre 8 chars.
f"{entier:08b}" — format directement sur 8 bits avec zéros.

✔ Correction — deux variantes

Variante 1 — avec .zfill(8) :

def str2qrcode(message):
    qrcode = []
    table_inverse = {valeur: cle for cle, valeur in dict_ascii.items()}
    for caractere in message:
        entier = table_inverse.get(caractere, 63)
        binaire_str = bin(entier)[2:].zfill(8)  # ← CORRECTION
        ligne = tuple(int(bit) for bit in binaire_str)
        qrcode.append(ligne)
    return qrcode

Variante 2 — avec f-string formatée (plus concise) :

def str2qrcode(message):
    qrcode = []
    table_inverse = {valeur: cle for cle, valeur in dict_ascii.items()}
    for caractere in message:
        entier = table_inverse.get(caractere, 63)
        binaire_str = f"{entier:08b}"       # ← CORRECTION : toujours 8 bits
        ligne = tuple(int(bit) for bit in binaire_str)
        qrcode.append(ligne)
    return qrcode

Vérification : str2qrcode("M.Hara") == figure1 doit valoir True après correction. Le format :08b signifie : binaire (b), sur 8 caractères (8), complété par des zéros à gauche (0).

Pièges fréquents & récapitulatif

Piège 1 — Oublier les zéros de tête dans bin()

bin(77) renvoie '0b1001101' → après [2:] : '1001101' (7 bits). Sans .zfill(8) le tuple n'a que 7 éléments, ce qui casse tout le pipeline. Toujours utiliser .zfill(8) ou f"{n:08b}".

Piège 2 — KeyError dans dec2str

La table ASCII standard couvre 0–127. Tout entier hors de cet intervalle (ex. 233 pour é) n'est pas dans dict_ascii et provoque un KeyError. Toujours utiliser .get(entier, "?") plutôt qu'un accès direct [entier].

Piège 3 — Confondre bin2dec et qrcode2dec

bin2dec travaille sur un seul tuple → renvoie un entier.
qrcode2dec travaille sur une liste de tuples → renvoie une liste d'entiers. Ne pas mélanger les deux niveaux d'abstraction.

Piège 4 — Horner vs puissances : ne pas inverser

Dans un QR code, le bit le plus à gauche est le bit de poids fort (2⁷). Si on itère de gauche à droite avec des puissances, il faut utiliser enumerate(reversed(t)) pour avoir les bons exposants. Avec Horner, on itère naturellement de gauche à droite sans se soucier des indices.

Récapitulatif — Ce que tu dois retenir
  • La conversion binaire → décimal se fait avec la méthode de Horner : r = r * 2 + bit.
  • qrcode2dec = appliquer bin2dec à chaque élément de la liste (compréhension de liste).
  • Pour les codes hors ASCII, utiliser dict.get(clé, valeur_défaut) pour éviter les KeyError.
  • Pour la conversion inverse, toujours forcer 8 bits avec .zfill(8) ou f"{n:08b}".
  • Le dictionnaire inverse se construit avec : {v: k for k, v in dict_ascii.items()}.
