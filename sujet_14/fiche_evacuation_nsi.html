1 — Rappels indispensables

Les tableaux 2D et leur parcours

La grille d'une pièce est une liste de listes en Python. grille[i][j] est la case à la ligne i, colonne j. Pour sommer tous les éléments, on utilise des boucles imbriquées ou des sommes imbriquées :

Trois façons équivalentes de sommer une grille 2D 
# Méthode 1 : boucles imbriquées
total = 0
for ligne in grille:
    for valeur in ligne:
        total += valeur

# Méthode 2 : sum imbriqué (plus compacte)
total = sum(sum(ligne) for ligne in grille)

# Méthode 3 : sum avec unpacking (avancée)
total = sum(v for ligne in grille for v in ligne)

La distance de Manhattan

Imagine une ville avec un réseau de rues en grille. Pour aller du point A au point B, tu ne peux pas couper en diagonale — tu dois emprunter les rues horizontales et verticales. La distance de Manhattan mesure ce trajet minimal.

Définition et calcul

La distance de Manhattan entre deux cases \((i_1, j_1)\) et \((i_2, j_2)\) est : $$d = |i_1 - i_2| + |j_1 - j_2|$$ 

def distance_de_manhattan(destination):
    """Fonction interne — i et j sont capturés depuis le contexte."""
    return abs(i - destination[0]) + abs(j - destination[1])

Exemple : depuis la case (3, 4), la sortie en (0, 5) est à \(|3-0| + |4-5| = 3 + 1 = 4\) cases. La sortie en (4, 1) est à \(|3-4| + |4-1| = 1 + 3 = 4\) cases également.

Le schéma algorithmique « trouver le minimum »

C'est exactement ce que doit faire choix_sortie : parcourir une liste et retenir l'élément qui minimise une valeur. Ce schéma revient souvent en NSI.

Patron — Trouver l'élément minimal 
meilleur     = liste[0]           # initialisation avec le premier élément
meilleure_val = critere(liste[0]) # valeur correspondante

for element in liste[1:]:         # parcours du reste
    val = critere(element)
    if val < meilleure_val:        # on a trouvé mieux
        meilleur     = element
        meilleure_val = val

return meilleur

Coordonnées des sorties — convention critique

Comment une sortie est représentée

Une sortie est un tuple (ligne, colonne) — dans le même format que les cases de la grille. Pour une pièce Piece(5, 7) :

DirectionTupleRaisonnement
"N" Nord(0, position)Ligne 0, colonne = position
"S" Sud(i_max, position)Dernière ligne, colonne = position
"O" Ouest(position, 0)Colonne 0, ligne = position
"E" Est(position, j_max)Dernière colonne, ligne = position

Mnémotechnique : N et S → même colonne, ligne extrême. O et E → même ligne, colonne extrême. La position est toujours l'indice parallèle au mur.

Question 2 — evacuation

Énoncé — Question 2

Q2 Écrire le corps de la fonction evacuation(p, silencieux=True) qui simule l'évacuation complète et renvoie le nombre de tours nécessaires. Elle doit appeler p.alerter(silencieux) à chaque tour, et afficher l'état de la pièce si silencieux=False.

Ce que fait alerter() — à comprendre absolument
  • p.alerter(silencieux) simule un tour : chaque occupant avance d'une case vers sa sortie.
  • Elle renvoie True si au moins un déplacement a eu lieu, False si personne n'a pu bouger.
  • Quand tous les occupants sont sortis, alerter() renvoie False (rien à déplacer).
  • On compte le nombre d'appels à alerter() = le nombre de tours.
Indice 1 
tours = 0
while p.alerter(silencieux):   # alerter() renvoie True tant qu'il y a des mouvements
    tours += 1
    if not silencieux:
        print(p)
return tours

Attention à l'ordre : on appelle alerter() dans la condition du while — l'appel et le test se font en même temps. On incrémente tours après chaque tour réussi.

Question 4 — Corriger les bugs de choix_sortie

Énoncé — Question 4

Q4 Identifier l'erreur logique et la variable non définie dans le code de choix_sortie, puis effectuer les corrections pour qu'elle renvoie la sortie la plus proche (au sens de la distance de Manhattan).

Voici le code fourni :

def choix_sortie(self, i: int, j: int) -> tuple:
    def distance_de_manhattan(destination):
        return abs(i - destination[0]) + abs(j - destination[1])
    assert len(self.sorties) > 0, "Aucune sortie"
    choix    = self.sorties[0]
    distance = distance_de_manhattan(choix)
    for k in range(1, len(self.sorties)):
        autre_sortie = self.sorties[k]
        if k < 0:                    ← BUG 1 : condition toujours fausse !
            choix    = autre_sortie
            distance = d2               ← BUG 2 : d2 n'est pas définie !
    return choix
Bug 1 — Condition logiquement impossible

for k in range(1, len(self.sorties)) fait varier k de 1 jusqu'à la longueur de la liste moins 1. k est donc toujours ≥ 1.

La condition if k < 0 est donc toujours fausse — le corps du if ne s'exécute jamais. La boucle tourne mais ne fait rien. La fonction renvoie toujours self.sorties[0], quelle que soit la position.

Ce qu'il fallait écrire : comparer la distance à la nouvelle sortie avec la distance au meilleur choix actuel : if distance_de_manhattan(autre_sortie) < distance

Bug 2 — Variable d2 non définie

Dans le corps du if, distance = d2 utilise d2 qui n'a jamais été initialisée. Python aurait levé une NameError si le if avait pu s'exécuter. La correction : calculer d2 = distance_de_manhattan(autre_sortie) avant le test, et utiliser cette variable dans la condition et l'affectation.

Résumé des deux corrections :

  • Ajouter d2 = distance_de_manhattan(autre_sortie) avant le test (correction du Bug 2 et préparation de la valeur à comparer).

Pièges fréquents & stratégie de performance

Piège 1 — Confondre (ligne, colonne) et (x, y)

Dans cette classe, toutes les positions sont des tuples (i, j) où i est la ligne (axe vertical) et j est la colonne (axe horizontal). Pour une sortie au Sud d'une pièce de profondeur 5 : la ligne est i_max = 4, pas j_max. Inverser les deux fait placer les sorties sur le mauvais mur.

Piège 2 — Oublier que alerter() s'appelle dans la condition du while

while p.alerter(silencieux): appelle alerter() et teste son résultat en même temps. Si tu écris d'abord p.alerter() puis while p.nb_occupants_restants() > 0, tu perdras un tour à chaque itération (un appel double) et le résultat sera erroné. La formulation en une seule ligne est la plus propre.

Piège 3 — Copier le code bugué de choix_sortie sans analyser

Le code fourni semble fonctionner (pas d'erreur au lancement) car la condition if k < 0 est simplement toujours fausse — Python ne plante pas. Mais le résultat est systématiquement faux : seule la première sortie est renvoyée. Il faut lire et analyser le code plutôt que de simplement l'exécuter. C'est exactement ce que Q4 demande.

Piège 4 — Ne pas identifier les deux bugs séparément

L'énoncé demande d'identifier l'erreur logique et la variable non définie — ce sont deux bugs distincts. À l'examen, mentionner les deux est indispensable pour avoir tous les points. Bug 1 = logique (condition impossible), Bug 2 = technique (NameError).
