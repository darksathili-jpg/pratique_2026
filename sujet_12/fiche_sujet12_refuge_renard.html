1 — Rappels : Programmation orientée objet (POO)

Imagine que tu veux modéliser un animal dans ton programme. Tu pourrais utiliser des variables séparées (nom = "Zorro", poids = 6.5…), mais dès que tu as 30 animaux, c'est le chaos. La POO résout ça : on crée un moule (la classe) qui décrit la forme de l'objet, puis on fabrique autant d'objets qu'on veut à partir de ce moule.

Définition – Classe et objet

Une classe est un modèle qui définit les attributs (données) et les méthodes (fonctions) d'un type d'objet. Un objet (ou instance) est un exemplaire concret créé à partir d'une classe.

La structure d'une classe Python

class Renard:
    def __init__(self, nom, poids):   # constructeur : appelé à la création
        self.nom   = nom              # attribut d'instance
        self.poids = poids

    def __str__(self):                # appelé par print() et str()
        return f"Renard : {self.nom}, {self.poids} kg"

    def est_adulte(self):             # méthode ordinaire
        return self.poids >= 6.0

# Créer un objet (instancier la classe) :
r = Renard("Zorro", 6.5)
print(r)            # → Renard : Zorro, 6.5 kg
print(r.nom)        # → Zorro
print(r.est_adulte()) # → True

Le rôle de self

self désigne l'objet lui-même. C'est le premier paramètre de toute méthode d'instance. Quand tu écris self.nom = nom dans __init__, tu crées un attribut attaché à cet objet précis, distinct de la variable locale nom.

Règle d'or

Dans __init__, chaque paramètre reçu doit être sauvegardé dans un attribut avec self.nom_attribut = parametre. Sinon, la valeur est perdue à la fin du constructeur.

Les méthodes spéciales __init__ et __str__

Méthodes spéciales (« dunder »)

__init__(self, ...) — Le constructeur. Python l'appelle automatiquement quand tu écris Renard(...). Il initialise les attributs de l'objet.

__str__(self) — Appelé automatiquement par print(objet) et str(objet). Doit retourner une chaîne de caractères (pas autre chose).

Schéma des deux classes du sujet

Renard
  • Attributs : identifiant (int), nom (str), poids (float), date_arrivee (str)
  • Méthodes : __init__, __str__
Refuge
  • Attributs : nom (str), adresse (str), liste_renards (list)
  • Méthodes : recueillir, lister_peu_corpulents, pourcentage_peu_corpulents, importer_donnees

2 — Rappels : lire un fichier CSV avec le module csv

Un fichier CSV (Comma-Separated Values) stocke des données tabulaires sous forme de texte brut, avec un séparateur entre les colonnes. Dans ce sujet, le séparateur est le point-virgule (;).

Le fichier donnees_renards.csv — extrait

idnompoidsdate_arrivee
101Zorro6.52023-01-15
102Roxy5.82023-02-10
103Filou7.22023-03-05
104Kira4.92024-11-20
… 26 autres lignes …
130Loup5.22023-09-01
Point crucial du sujet

Toutes les données lues depuis un fichier CSV arrivent sous forme de chaînes de caractères (str), même si la valeur ressemble à un nombre. Il faut convertir explicitement id en int et poids en float. C'est exactement le bug de la Question 3.

Comment fonctionne csv.DictReader

import csv

with open('fichier.csv', 'r', encoding='utf-8') as f:
    lignes = csv.DictReader(f, delimiter=';')
    for ligne in lignes:
        # 'ligne' est un dictionnaire {colonne: valeur}
        # TOUTES les valeurs sont des str !
        print(ligne['id'])     # → '101'  (str, pas int)
        print(ligne['poids'])  # → '6.5'  (str, pas float)

Pour convertir, on utilise int(ligne['id']) et float(ligne['poids']).

5 — Q3 Identifier et corriger le bug de importer_donnees

Énoncé de la question 3

L'exécution de la méthode importer_donnees provoque une erreur logique lors de la manipulation ultérieure du poids et de l'identifiant des renards. Identifier la source de cette erreur, proposer une correction, puis tester en instanciant le refuge « SOS Goupil » et en y important le fichier donnees_renards.csv.

Analyse du bug

Voici la ligne problématique dans la méthode fournie : 

renard = Renard(ligne['id'], ligne['nom'], ligne['poids'], ligne['date_arrivee'])
#               ↑ str !      ↑ str ok     ↑ str !          ↑ str ok

csv.DictReader lit tout comme du texte. La valeur ligne['id'] vaut la chaîne '101', pas l'entier 101. De même, ligne['poids'] vaut '6.5', pas le flottant 6.5.

Erreur logique — pas d'erreur immédiate !

C'est une erreur logique, pas une erreur de syntaxe. Python ne se plaint pas quand tu passes une str là où on attend un int — il crée l'objet sans sourciller. L'erreur n'apparaît que plus tard, quand tu tentes une opération numérique sur le poids (comme comparer renard.poids < 6.0 entre une str et un float). 

# Exemple de ce qui se passe sans la correction :
>>> '5.8' < 6.0   # TypeError : impossible de comparer str et float !
# Ou pire :
>>> '5.8' + 0.5   # TypeError

La correction

Il faut convertir explicitement id en int et poids en float au moment de la création de l'objet :

Indice — syntaxe de conversion 
int('101')    # → 101    (str vers int)
float('6.5')  # → 6.5    (str vers float)
float('5.8')  # → 5.8    (str vers float)

Éditeur Python — à toi de jouer !

Correction – Question 3

Ligne buguée :

renard = Renard(ligne['id'], ligne['nom'], ligne['poids'], ligne['date_arrivee'])

Ligne corrigée :

renard = Renard(int(ligne['id']), ligne['nom'], float(ligne['poids']), ligne['date_arrivee'])

Seuls id (converti en int) et poids (converti en float) nécessitent une conversion. nom et date_arrivee sont déjà des chaînes — pas de conversion nécessaire.

Méthode corrigée complète :

def importer_donnees(self, nom_fichier):
    print(f"Tentative d'importation depuis {nom_fichier}...")
    with open(nom_fichier, 'r', encoding='utf-8') as f:
        lignes = csv.DictReader(f, delimiter=';')
        for ligne in lignes:
            renard = Renard(
                int(ligne['id']),
                ligne['nom'],
                float(ligne['poids']),
                ligne['date_arrivee']
            )
            self.recueillir(renard)

7 — Pièges fréquents & astuces de vérification

Piège 1 — Oublier self. devant un attribut

Sans self., tu crées une variable locale qui disparaît à la fin de la méthode. L'attribut n'est pas créé sur l'objet. 

# ❌ Mauvais : identifiant est une variable locale, perdue après __init__
def __init__(self, identifiant, ...):
    identifiant = identifiant   # sans self.

# ✓ Correct
def __init__(self, identifiant, ...):
    self.identifiant = identifiant
Piège 2 — __str__ qui print au lieu de return

__str__ doit retourner une chaîne, pas l'afficher. Si tu fais print(...) à l'intérieur, Python reçoit None comme valeur de retour — et print(renard) affiche None. 

# ❌ Mauvais
def __str__(self):
    print(f"Renard ID {self.identifiant} ...")   # print dans __str__ → None !

# ✓ Correct
def __str__(self):
    return f"Renard ID {self.identifiant} ..."   # return une str
Piège 3 — Types CSV : oublier int() ou float()

C'est LE piège central du sujet. Toutes les valeurs lues par csv.DictReader sont des str. Sans conversion, la comparaison renard.poids < 6.0 lève une TypeError. 

# ❌ Mauvais
Renard(ligne['id'], ligne['nom'], ligne['poids'], ligne['date_arrivee'])
#      str !                      str !

# ✓ Correct
Renard(int(ligne['id']), ligne['nom'], float(ligne['poids']), ligne['date_arrivee'])
Piège 4 — Condition stricte : poids < 6.0, pas ≤

L'énoncé précise « strictement inférieur à 6,0 kg ». Un renard pesant exactement 6,0 kg n'est pas peu corpulent. Vérifie que tu utilises < et non <=. 

renard.poids < 6.0   # ✓ strictement inférieur
renard.poids <= 6.0  # ❌ inclut 6.0 — faux ici
Piège 5 — Confondre la méthode et son appel

Une méthode se définit avec def methode(self) et s'appelle avec objet.methode() (avec les parenthèses). Oublier les parenthèses retourne la méthode elle-même, pas son résultat. 

mon_refuge.lister_peu_corpulents    # ❌ retourne l'objet-méthode
mon_refuge.lister_peu_corpulents()  # ✓ retourne la liste
