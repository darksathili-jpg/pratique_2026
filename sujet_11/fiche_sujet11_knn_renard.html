1 — Rappels : dictionnaires, tuples et listes

Avant de plonger dans le sujet, faisons un point rapide sur les structures de données que tu vas manipuler à longueur de questions.

Les dictionnaires

Un dictionnaire Python associe des clés à des valeurs. Dans ce sujet, chaque zone géographique est représentée par un dictionnaire à cinq clés : 

zone = {
    'vegetation':            9,   # note de 0 à 10
    'proximite_eau':         6,   # note de 0 à 10
    'densite_urbaine':       0,   # note de 0 à 10
    'disponibilite_proies':  4,   # note de 0 à 10
    'presence_renard':       1    # 1 si oui, 0 si non
}

Pour accéder à une valeur, tu écris zone['vegetation']. Pour modifier une valeur, tu écris zone['presence_renard'] = 0. Tu peux aussi parcourir toutes les clés avec for cle in zone: ou obtenir les paires clé-valeur avec zone.items().

Astuce clé du sujet

La clé 'presence_renard' vaut 1 (renard présent) ou 0 (renard absent). Ce n'est pas un booléen, c'est un entier. Quand tu veux tester la présence, compare bien : if habitat['presence_renard'] == 1:, pas if habitat['presence_renard']: (même si ça marche en Python, c'est une source d'erreur à l'examen).

Les tuples

Un tuple est une paire (ou triplet, quadruplet…) de valeurs regroupées ensemble, notées entre parenthèses. Contrairement à une liste, on ne peut pas modifier un tuple après sa création. 

t = (7.21, {'vegetation': 9, 'proximite_eau': 6, ...})
# Accéder au premier élément (la distance) :
t[0]   # → 7.21
# Accéder au second élément (le dictionnaire) :
t[1]   # → {'vegetation': 9, ...}

Dans ce sujet, tu vas beaucoup manipuler des listes de tuples : chaque tuple contient une distance et le dictionnaire de la zone correspondante.

Définition – tuple

Un tuple est une séquence ordonnée et immuable. On accède à ses éléments avec leur indice : t[0] pour le premier, t[1] pour le second, etc. On crée un tuple avec des parenthèses : (distance, habitat).

Trier une liste avec une clé personnalisée

La fonction liste.sort(key=...) trie une liste selon un critère que tu choisis. Quand tu as une liste de tuples [(7.21, {...}), (5.19, {...}), ...] et que tu veux la trier selon la distance (le premier élément de chaque tuple), tu utilises : 

distances.sort(key=lambda x: x[0])

Le lambda x: x[0] est une petite fonction anonyme qui dit « prends l'élément x et renvoie son premier composant ». C'est déjà écrit dans le code fourni — tu n'as pas à l'écrire toi-même.

2 — L'algorithme des k plus proches voisins

Imagine que tu cherches à savoir si un renard pourrait vivre dans une nouvelle zone de forêt. Tu ne peux pas le savoir à l'avance, mais tu as déjà un grand tableau de zones déjà connues avec la réponse (renard présent ou non). L'idée est simple :

Idée intuitive

« Dis-moi qui sont tes voisins, je te dirai qui tu es. »
Si les zones les plus ressemblantes à ta nouvelle zone accueillent toutes des renards, il y a de bonnes chances que ta zone en accueille aussi.

La question est donc : comment mesure-t-on la « ressemblance » entre deux zones ? En calculant une distance. Plus deux zones ont des caractéristiques proches (beaucoup de végétation, eau disponible, peu de présence humaine…), plus leur distance est petite.

Visualisation : qui sont les k=3 voisins les plus proches du point bleu ?

🦊 🌿 🦊 ❓ 🦊 🌿 🌿

3 voisins les plus proches du ❓ → 2 renards 🦊, 1 sans renard 🌿 → majorité = présence probable ✓

Les étapes de l'algorithme k-NN

Algorithme – k plus proches voisins
  1. Calculer la distance entre le nouvel habitat et chacun des habitats connus.
  2. Trier ces distances de la plus petite à la plus grande.
  3. Garder les k premiers (les k plus proches voisins).
  4. Voter : si plus de k/2 voisins ont un renard → présence probable.

La formule de distance entre deux habitats

Pour mesurer à quel point deux habitats se ressemblent, on utilise une formule mathématique proche de la distance « à vol d'oiseau » dans un espace à 4 dimensions (une par caractéristique).

Si l'habitat A a les valeurs \((v_a, p_a, de_a, di_a)\) et l'habitat B a les valeurs \((v_b, p_b, de_b, di_b)\), alors :

$$d = \sqrt{(v_a - v_b)^2 + (p_a - p_b)^2 + (de_a - de_b)^2 + (di_a - di_b)^2}$$

En Python, la racine carrée s'obtient avec sqrt(...) du module math. Pour calculer chaque terme, tu fais (habitat_1['vegetation'] - habitat_2['vegetation'])**2, etc.

3 — Q1 Écrire la fonction distance

Énoncé de la question 1

Écrire le code de la fonction distance qui prend en paramètres deux habitats sous la forme de dictionnaires et renvoie la valeur « distance » entre ces deux habitats définie ci-dessus. On rappelle que la racine carrée se calcule avec la fonction sqrt du module math.

Aide méthodologique

La formule te demande de faire la somme de cinq termes, chacun étant le carré d'une différence. Voici la marche à suivre :

  1. Pour chaque clé ('vegetation', 'proximite_eau', 'densite_urbaine', 'disponibilite_proies', calcule la différence habitat_1[cle] - habitat_2[cle].
  2. Mets cette différence au carré : ** 2.
  3. Additionne ces quatre carrés.
  4. Prends la racine carrée du tout avec sqrt(...).
Indice niveau 1

Structure générale de la fonction : 

def distance(habitat_1, habitat_2):
    somme = (habitat_1['vegetation'] - habitat_2['vegetation'])**2 \
          + ...   # continue pour les 3 autres clés
    return sqrt(somme)
Indice niveau 2 — version avec boucle 
def distance(habitat_1, habitat_2):
    cles = ['vegetation', 'proximite_eau',
            'densite_urbaine', 'disponibilite_proies']
    somme = 0
    for cle in cles:
        diff = habitat_1[cle] - habitat_2[cle]
        somme += diff ** 2
    return sqrt(somme)

Énoncé de la question 2

Écrire le code de la fonction distance_d_un_habitat qui prend en paramètres un habitat (dictionnaire) et une liste d'habitats (liste de dictionnaires). La fonction doit renvoyer une liste de tuples où chaque tuple contient : la distance entre l'habitat fourni et un habitat de la liste ; puis le dictionnaire représentant cet habitat.

Aide méthodologique

Cette fonction fait appel à la fonction distance que tu viens d'écrire. L'idée : parcourir chaque habitat de la liste, calculer sa distance avec l'habitat de référence, et stocker le résultat sous forme de tuple.

  1. Crée une liste vide pour accumuler les résultats.
  2. Parcours chaque zone de la liste habitats.
  3. Calcule d = distance(habitat, zone).
  4. Ajoute le tuple (d, zone) à ta liste.
  5. Renvoie la liste complète.
Indice niveau 1 
def distance_d_un_habitat(habitat, habitats):
    resultats = []
    for zone in habitats:
        d = distance(habitat, zone)
        resultats.append((d, zone))  # un tuple !
    return resultats
Indice niveau 2 — version compréhension de liste 
def distance_d_un_habitat(habitat, habitats):
    return [(distance(habitat, zone), zone) for zone in habitats]

Les deux versions sont équivalentes. La première est plus lisible pour l'examen.

6 — Q4 Corriger presence_renard

Énoncé de la question 4

La fonction presence_renard contient une erreur de traitement des tuples. Corriger la fonction presence_renard.

Analyse du bug – apprends à lire du code bogué

Voici le code fourni dans l'énoncé, avec le bug en place. Lis-le attentivement ligne par ligne : 

def presence_renard(k, habitat, habitats):
    habitats = k_plus_proches(k, habitat, habitats)
    n_renards = 0
    for habitat in habitats:
        distance = habitat[0]
        caracteristiques = habitat[1]
        if distance['presence_renard'] == 1:  # ← ICI le bug !
            n_renards += 1
    return n_renards > k/2

Repère la ligne if distance['presence_renard'] == 1:. Réfléchis : distance vaut habitat[0], c'est-à-dire le premier élément du tuple. Or habitat[0] est un nombre flottant (la distance euclidienne), pas un dictionnaire !

Erreur identifiée

Le code utilise distance['presence_renard'] mais distance est un float. On ne peut pas indexer un float avec une clé comme si c'était un dictionnaire. Ce qu'on voulait tester, c'est caracteristiques['presence_renard'], car c'est caracteristiques qui pointe vers le dictionnaire de la zone.

Le correctif est simple : remplacer distance['presence_renard'] par caracteristiques['presence_renard'].

Indice – la structure d'un tuple de résultat

Un tuple de la liste renvoyée par k_plus_proches ressemble à :

tuple_exemple = (5.196, {'vegetation': 8, ..., 'presence_renard': 0})
#                 ↑ indice 0 : float        ↑ indice 1 : dict

Donc habitat[0] → float (la distance), habitat[1] → dict (les caractéristiques).

7 — Q5 Le nouvel habitat est-il habitable ?

Énoncé de la question 5

L'habitat nouveau est-il susceptible ou non de contenir une population de renards ? Expliquer en utilisant la fonction précédente avec plusieurs valeurs pour k.

Aide méthodologique

Cette question est une question d'interprétation. Il ne s'agit pas d'écrire du code complexe, mais de tester la fonction presence_renard pour différentes valeurs de k et d'analyser les résultats.

Méthodologie recommandée pour l'examen
  1. Tester avec plusieurs valeurs impaires de k (1, 3, 5, 7…) — impaires pour éviter les égalités.
  2. Pour chaque valeur, noter si la fonction renvoie True ou False.
  3. Conclure en observant la tendance majoritaire.
  4. Rédiger une phrase de conclusion claire pour l'examinateur.
Remarque sur le choix de k

En général, on préfère utiliser des valeurs impaires de k pour éviter les ex-æquo dans le vote. Si k/2 n'est pas entier, Python gère correctement la comparaison n_renards > k/2. Pour k=4, par exemple, k/2=2.0, et il faut donc strictement plus de 2 renards (soit au moins 3).

8 — Pièges fréquents & astuces de vérification

Piège 1 — Confondre les deux éléments d'un tuple

C'est exactement le bug de la Question 4. Un résultat de k_plus_proches est une liste de tuples (distance, dictionnaire). L'indice [0] donne le float, l'indice [1] donne le dict. Ne jamais confondre les deux. 

# ❌ Mauvais
if tuple_voisin[0]['presence_renard'] == 1:   # [0] est un float !

# ✓ Correct
if tuple_voisin[1]['presence_renard'] == 1:   # [1] est le dict
Piège 2 — Oublier sqrt ou mal importer

La racine carrée sqrt vient du module math. Pense à écrire en tout début de fichier : from math import sqrt. Sans cet import, Python lèvera une erreur NameError: name 'sqrt' is not defined.

Piège 3 — Ne pas inclure 'presence_renard' dans la distance

La formule de distance du sujet utilise les 5 clés, y compris 'presence_renard'. Beaucoup d'élèves n'utilisent que les 4 premières (végétation, eau, densité, proies) par intuition — c'est faux ici. Suis la formule à la lettre.

Piège 4 — Modifier la variable de boucle par erreur

Dans la fonction presence_renard d'origine, la variable de boucle est nommée habitat — le même nom que le paramètre de la fonction ! Cela écrase le paramètre dans la boucle. Pour éviter ce genre d'ambiguïté, utilise un nom différent comme voisin. 

# ❌ Risqué (écrase le paramètre 'habitat' du haut)
for habitat in voisins:  ...

# ✓ Plus clair
for voisin in voisins:  ...
Piège 5 — Mal comprendre la condition > k/2

La condition n_renards > k/2 signifie « strictement plus de la moitié ». Si k=4 et n_renards=2, le résultat est False (car 2 n'est pas strictement supérieur à 2). Pour que ce soit True avec k=4, il faut 3 renards ou plus.
