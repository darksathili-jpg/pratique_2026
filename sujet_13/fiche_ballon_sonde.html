1 — Rappels essentiels

Dépackager le résultat d'une fonction qui renvoie plusieurs valeurs

En Python, une fonction peut renvoyer plusieurs valeurs séparées par des virgules. À l'appel, tu les récupères dans autant de variables que de valeurs renvoyées — c'est le dépackage. 

def deux_valeurs():
    return 10, 20

a, b = deux_valeurs()   # a = 10,  b = 20
print(a, b)             # → 10 20

Dans ce sujet, recupere_donnees_fichier_csv renvoie 4 listes dans cet ordre : altitudes, temperatures, longitudes, latitudes.

Parcourir une liste et construire une nouvelle liste

# Version boucle classique
def convertir(liste):
    resultat = []
    for valeur in liste:
        resultat.append(valeur - 273.15)
    return resultat

# Version compréhension de liste (équivalente, plus concise)
def convertir(liste):
    return [valeur - 273.15 for valeur in liste]

Trouver le minimum d'une liste

t_min = min(liste_temperatures)  # renvoie la plus petite valeur

L'assertion — garantir une condition

Une assertion vérifie qu'une condition est vraie. Si elle est fausse, Python lève une AssertionError et stoppe l'exécution. C'est un outil de défense qu'on place au début d'une fonction pour s'assurer que les données d'entrée sont cohérentes. 

assert condition, "Message d'erreur optionnel"

# Exemple :
assert len(liste_a) == len(liste_b), "Les deux listes doivent avoir la même longueur"
Conversion Kelvin ↔ Celsius

La formule est : \(T_{°C} = T_K - 273{,}15\). Donc on soustrait 273,15. Inversement, \(T_K = T_{°C} + 273{,}15\). À 0°C correspond 273,15 K. Les kelvins ne sont jamais négatifs.

5 — Q3 Écrire altitude_la_plus_froide

Énoncé de la question 3

Proposer une écriture de la fonction altitude_la_plus_froide qui prend en paramètres une liste d'altitudes et une liste de températures en °C, et renvoie un tuple : (température minimale, liste des altitudes correspondantes).

Si la température minimale est atteinte à plusieurs altitudes, toutes doivent apparaître dans la liste.

Les exemples de l'énoncé

>>> altitudes   = [7000, 10125, 13896, 14211]
>>> temperatures = [-35.2, -52.1, -57.4, -57.4]
>>> altitude_la_plus_froide(altitudes, temperatures)
(-57.4, [13896, 14211])       # deux altitudes à égalité

>>> altitudes   = [6000, 7250, 11542, 15214, 17300]
>>> temperatures = [-33.7, -45, -53, -58.5, -60.1]
>>> altitude_la_plus_froide(altitudes, temperatures)
(-60.1, [17300])              # une seule altitude

Aide méthodologique

  1. Trouver la température minimale de la liste avec min().
  2. Parcourir simultanément les deux listes (avec zip ou un indice i).
  3. Chaque fois que la température vaut le minimum, ajouter l'altitude correspondante à une liste.
  4. Retourner un tuple (t_min, liste_altitudes).
Indice niveau 1 – structure avec zip 
def altitude_la_plus_froide(liste_altitudes, liste_temperatures):
    t_min = min(liste_temperatures)
    alts_min = []
    for alt, temp in zip(liste_altitudes, liste_temperatures):
        if temp == t_min:
            alts_min.append(alt)
    return (t_min, alts_min)
Indice niveau 2 – structure avec indice 
def altitude_la_plus_froide(liste_altitudes, liste_temperatures):
    t_min = min(liste_temperatures)
    alts_min = []
    for i in range(len(liste_altitudes)):
        if liste_temperatures[i] == t_min:
            alts_min.append(liste_altitudes[i])
    return (t_min, alts_min)

Résultat exact sur releves_ballon_sonde.csv

Aide méthodologique

La boucle for i in range(len(liste_longitudes)) dans genere_kml utilise l'indice i pour accéder aux deux listes simultanément. Si les deux listes n'ont pas la même longueur, on risque un IndexError (accès hors limites). L'assertion, placée au début de la fonction, stoppe l'exécution proprement avant que ce problème arrive.

assert len(liste_longitudes) == len(liste_latitudes), \

Juste après la ligne def genere_kml(liste_longitudes, liste_latitudes):, avant toute autre instruction. 

def genere_kml(liste_longitudes, liste_latitudes):
    assert len(liste_longitudes) == len(liste_latitudes)  # ← ICI
    fichier_kml = open('ballon sonde.kml', 'w')
    ...

8 — Q6 Corriger le bug de la balise </kml>

Énoncé de la question 6

La balise fermante </kml> est absente en toute fin de fichier, ce qui cause des problèmes de compatibilité. Proposer une amélioration du code visant à répondre à cette difficulté.

Analyse de la fonction fournie

Observe la fin de la fonction genere_kml telle qu'elle est fournie : 

    bas_fichier = '\n'
    fichier_kml.write(bas_fichier)
    fichier_kml.close()          # ← le fichier est fermé ICI

Le fichier KML valide doit se terminer par : 

...
</Document>
</kml>     ← cette ligne est absente !
Structure d'un fichier KML valide 
<?xml version="1.0" encoding="UTF-8"?>
<kml xmlns="http://www.opengis.net/kml/2.2">   ← ouvre kml
<Document>
  <Placemark> ... </Placemark>
  ...
</Document>
</kml>                                          ← ferme kml (MANQUANT)

La correction

Il suffit d'ajouter l'écriture de </kml>\n dans bas_fichier avant la fermeture du fichier. Deux solutions équivalentes :

Indice – deux approches possibles 
# Solution A : modifier bas_fichier
bas_fichier = '</Document>\n'
bas_fichier += '</kml>\n'      # ← ajouter cette ligne
fichier_kml.write(bas_fichier)
fichier_kml.close()

# Solution B : écrire séparément
bas_fichier = '</Document>\n'
fichier_kml.write(bas_fichier)
fichier_kml.write('</kml>\n')  # ← ajouter cette ligne
fichier_kml.close()

Éditeur Python — à toi de jouer !

Correction – Question 6 
# Dans genere_kml, remplacer :
bas_fichier = '</Document>\n'
fichier_kml.write(bas_fichier)
fichier_kml.close()

# Par :
bas_fichier = '</Document>\n'
bas_fichier += '</kml>\n'      # ← correction : ajout de la balise fermante
fichier_kml.write(bas_fichier)
fichier_kml.close()

La balise </kml> doit être après </Document> et avant la fermeture du fichier. L'ordre est impératif dans un document XML/KML.

9 — Pièges fréquents & astuces de vérification

Piège 1 — Inverser l'ordre des 4 listes au dépackage (Q1)

La fonction renvoie dans l'ordre : altitudes, temperatures, longitudes, latitudes. Inverser deux variables (par exemple mettre latitudes avant longitudes) ne produit aucune erreur Python, mais les données sont mélangées silencieusement. 

# ❌ Mauvais : l'ordre ne correspond pas à la fonction
longitudes, latitudes, altitudes, temperatures = recupere_donnees_fichier_csv(...)

# ✓ Correct : même ordre que le return de la fonction
altitudes, temperatures, longitudes, latitudes = recupere_donnees_fichier_csv(...)
Piège 2 — Oublier round() dans conversion_K_en_C (Q2)

Sans round(..., 1), les résultats de 288.15 - 273.15 peuvent donner 14.999999999999972 au lieu de 15.0 à cause des flottants Python. L'arrondi est obligatoire pour passer les assertions. 

# ❌ Sans arrondi : erreur de précision flottante possible
[t - 273.15 for t in liste]

# ✓ Avec arrondi
[round(t - 273.15, 1) for t in liste]
Piège 3 — Ne pas collecter toutes les altitudes à égalité (Q3)

Si on s'arrête à la première altitude trouvée (return dans la boucle), on rate les altitudes suivantes qui ont la même température minimale. La boucle doit aller jusqu'au bout. 

# ❌ Mauvais : sort à la première occurrence
for alt, temp in zip(alt, temp_C):
    if temp == t_min:
        return (t_min, [alt])   # manque les autres !

# ✓ Correct : collecte tout, renvoie après
alts_min = []
for alt, temp in zip(alt, temp_C):
    if temp == t_min:
        alts_min.append(alt)
return (t_min, alts_min)
Piège 4 — Placer l'assertion après les premiers calculs (Q4)

L'assertion doit être la première instruction de la fonction, avant toute écriture dans le fichier. Si elle est placée au milieu, le fichier KML peut être partiellement écrit avant que l'erreur soit détectée.

Piège 5 — Inverser longitudes et latitudes (Q5)

La signature de genere_kml est (liste_longitudes, liste_latitudes). Si tu appelles genere_kml(latitudes, longitudes), le fichier KML sera généré avec des coordonnées inversées — les points seront mal placés sur la carte.

Piège 6 — Oublier fichier_kml.close() après la correction (Q6)

La ligne fichier_kml.close() doit rester en toute fin de fonction. Si elle est supprimée ou oubliée, le fichier reste ouvert et les données peuvent ne pas être entièrement écrites sur le disque.

  • La dernière ligne est </kml> (après Q6)
