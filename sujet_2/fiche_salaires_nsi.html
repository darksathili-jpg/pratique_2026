1 — Rappels : les listes de dictionnaires

Imagine que tu dois gérer un annuaire d'élèves dans ton lycée. Chaque élève a un prénom, une note de maths et une note de NSI. Comment stocker tout ça en Python ? La réponse la plus naturelle : une liste de dictionnaires.

Définition – Liste de dictionnaires

Une liste de dictionnaires est une liste Python dont chaque élément est un dictionnaire. Tous les dictionnaires partagent en général les mêmes clés — un peu comme les colonnes d'un tableau. Chaque dictionnaire représente une « ligne » de ce tableau.

Par exemple, voici un petit annuaire de trois élèves : 

annuaire = [
    {'prenom': 'Alice', 'maths': 17, 'nsi': 18},
    {'prenom': 'Bob',   'maths': 12, 'nsi': 15},
    {'prenom': 'Carla', 'maths': 14, 'nsi': 16}
]

Pour accéder à la note de NSI d'Alice, tu écris annuaire[0]['nsi']. Pour parcourir tous les élèves et afficher leurs prénoms : 

for eleve in annuaire:
    print(eleve['prenom'])
Astuce – Filtrer une liste

Pour garder uniquement les élèves ayant plus de 15 en NSI, tu peux utiliser une compréhension de liste : 

bons_en_nsi = [e for e in annuaire if e['nsi'] > 15]
# résultat : [{'prenom': 'Alice', ...}, {'prenom': 'Carla', ...}]

C'est exactement ce principe que tu vas utiliser tout au long de ce sujet : filtrer les employés selon leur sexe, leur expérience ou leur niveau d'études.

Calculer la moyenne d'un champ

Pour obtenir la moyenne de toutes les notes de NSI, deux approches classiques : 

# Approche 1 : boucle for
total = 0
for e in annuaire:
    total = total + e['nsi']
moyenne = total / len(annuaire)

# Approche 2 : avec sum() et une compréhension (plus compact)
moyenne = sum(e['nsi'] for e in annuaire) / len(annuaire)
Rappel – Renvoyer None

Certaines fonctions doivent renvoyer None quand le résultat n'a pas de sens — par exemple la moyenne d'une liste vide (division par zéro !). En Python, on teste cela avec if len(liste) == 0: return None ou plus simplement if not liste: return None.

Question 1 — Salaire moyen conditionnel

Énoncé — Question 1

Q1 Écrire le corps de la fonction salaire_moyen_condition(employes, champ, valeur) qui renvoie le salaire moyen des employés dont la valeur associée au champ champ est égale à valeur. La fonction renvoie None si aucun employé ne correspond.

Exemple : salaire_moyen_condition(employes, 'sexe', 'F') renvoie 2400.0. Ensuite, utiliser le jeu complet pour calculer les salaires moyens des femmes et des hommes.

Aide méthodologique

Décompose le problème en trois étapes simples :

  1. Filtrer : garde uniquement les employés dont emp[champ] == valeur.
  2. Vérifier : si la liste filtrée est vide, renvoie None.
  3. Calculer : renvoie sum(...) / len(...) sur les salaires filtrés.
Exemple chiffré – vérification manuelle

Sur le jeu de test, les 3 femmes ont des salaires 2 400, 2 500 et 2 300. Leur moyenne vaut \(\frac{2400 + 2500 + 2300}{3} = \frac{7200}{3} = 2400{,}0\).

Les 3 hommes ont des salaires 2 550, 2 800 et 2 700. Leur moyenne vaut \(\frac{2550 + 2800 + 2700}{3} = \frac{8050}{3} \approx 2683{,}3\).

Indice 1

Pour obtenir uniquement les salaires des employés vérifiant la condition : 

salaires_filtres = [emp['salaire'] for emp in employes if emp[champ] == valeur]
Indice 2

Après avoir filtré, teste si la liste est vide avant de diviser : 

if len(salaires_filtres) == 0:
    return None
return sum(salaires_filtres) / len(salaires_filtres)

Correction — Question 1

La clé de cette fonction : utiliser emp[champ] (le champ est une variable) plutôt que d'écrire emp['sexe'] en dur — ce qui la rendrait générique.

Sur le jeu complet (2 000 employés) :
Salaire moyen des femmes ≈ 2 229 €
Salaire moyen des hommes ≈ 2 438 €

On observe déjà un écart d'environ 200 € — soit près de 8,6 %. C'est ce que la Question 3 te fera calculer précisément.

Question 2 — Effectifs par sexe

Énoncé — Question 2

Q2 Écrire la fonction effectif_par_sexe(employes) qui prend en paramètre un tableau non vide d'employés et qui renvoie un dictionnaire ayant deux clés 'F' et 'M' associées respectivement au nombre de femmes et au nombre d'hommes.

Exemple : effectif_par_sexe(employes) renvoie {'F': 3, 'M': 3}.

Aide méthodologique

Deux approches sont envisageables. Choisis celle qui te semble la plus lisible.

Approche A — Compteur explicite 
nb_f = 0
nb_m = 0
for emp in employes:
    if emp['sexe'] == 'F':
        nb_f += 1
    else:
        nb_m += 1
return {'F': nb_f, 'M': nb_m}
Approche B — Avec sum() et une compréhension 
nb_f = sum(1 for emp in employes if emp['sexe'] == 'F')
nb_m = sum(1 for emp in employes if emp['sexe'] == 'M')
return {'F': nb_f, 'M': nb_m}
Indice

Sur le jeu de test, les lignes 1, 3 et 5 sont des femmes (3 au total), les lignes 2, 4 et 6 sont des hommes (3 au total). Le résultat attendu est donc {'F': 3, 'M': 3}.

Éditeur Python — Complète la fonction
Correction — Question 2

Note que la fonction renvoie bien {'F': 0, 'M': 1} (et non pas {'M': 1}) quand il n'y a que des hommes — les deux clés sont toujours présentes.

Question 3 — Déboguer calcul_ecart_sexe

Énoncé — Question 3

Q3 La fonction calcul_ecart_sexe du fichier analyse.py est incorrecte. Explique pourquoi, propose des assertions pour mettre les erreurs en évidence, puis écris une version corrigée nommée ecart_salaire. Calcule enfin l'écart en pourcentage sur les 2 000 employés.

Code bugué original

def calcul_ecart_sexe(employes):
    '''Renvoie l'écart de salaire en pourcentage pour les femmes
    par rapport aux hommes'''
    moy_h = salaire_moyen_condition(employes, 'sexe', 'M')
    moy_f = salaire_moyen_condition('employes', 'sexe', 'F')  # BUG 1
    return moy_h - moy_f                                   # BUG 2

Analyse des deux bugs

Bug 1 — Chaîne de caractères à la place de la variable

Regarde attentivement : salaire_moyen_condition('employes', 'sexe', 'F'). Le premier argument est la chaîne 'employes' (entre guillemets) au lieu de la variable employes (sans guillemets).

Résultat : Python va itérer sur les caractères de la chaîne 'employes' au lieu de parcourir la liste d'employés. Aucun caractère n'a le champ 'sexe', donc la fonction renvoie None — et la soustraction moy_h - None lève une TypeError.

Bug 2 — Différence au lieu du pourcentage

La formule correcte de l'écart en pourcentage est :

\(\text{écart} = \dfrac{\text{salaire moyen hommes} - \text{salaire moyen femmes}}{\text{salaire moyen hommes}} \times 100\)

Or la fonction renvoie moy_h - moy_f — une différence en euros, pas un pourcentage. De plus, si moy_h ou moy_f est None (un seul sexe présent), la soustraction plantera.

Assertions de mise en évidence

Voici des assertions qui permettent de détecter les deux bugs : 

# Bug 1 : sur le jeu de test, moy_f ne devrait pas être None
# (mais avec le bug, 'employes' est une chaine -> None -> TypeError)

# Bug 2 : l'écart sur le jeu de test doit être un pourcentage (entre 0 et 100)
# et non une difference en euros (~283 €)

employes_only_f = [e for e in employes if e['sexe'] == 'F']
# Si un seul sexe est présent, ecart_salaire doit renvoyer None
assert ecart_salaire(employes_only_f) == None

# L'ecart en pourcentage doit être compris entre 0 et 100
assert 0 <= ecart_salaire(employes) <= 100
Indice

Décompose la fonction corrigée en 4 étapes :

  1. Calcule moy_h = salaire_moyen_condition(employes, 'sexe', 'M').
  2. Calcule moy_f = salaire_moyen_condition(employes, 'sexe', 'F') (sans guillemets !).
  3. Si moy_h est None ou moy_f est None, renvoie None.
  4. Sinon renvoie (moy_h - moy_f) / moy_h * 100.
Éditeur Python — Écris la version corrigée
Correction — Question 3

Résumé des deux corrections :

  • Ligne 2 : 'employes' → employes (supprimer les guillemets).
  • Ligne 3 : ajouter un garde if moy_h is None or moy_f is None: return None.
  • Formule : remplacer moy_h - moy_f par (moy_h - moy_f) / moy_h * 100.

Sur le jeu de test : écart ≈ 10,6 %
Sur les 2 000 employés : écart ≈ 8,6 %

On retrouve un ordre de grandeur proche des statistiques réelles françaises (environ 15 % selon l'INSEE pour l'ensemble du marché du travail, mais variable selon les secteurs).

Question 4 — Algorithme des k plus proches voisins

Énoncé — Question 4

Q4 L'entreprise utilise l'algorithme des k plus proches voisins pour proposer un salaire d'embauche. Tester salaire_par_proximite sur deux profils identiques (même expérience, mêmes études) mais de sexe différent. Identifier la source de l'écart dans le programme et la corriger.

Principe de l'algorithme des k plus proches voisins

Imagine que tu arrives dans une nouvelle ville et que tu cherches un appartement. Pour estimer le loyer raisonnable, tu regardes les 3 appartements les plus similaires au tien (même surface, même quartier) et tu fais la moyenne de leurs loyers. C'est exactement l'idée des k plus proches voisins (k-NN).

Algorithme k-NN – Étapes
  1. Définir une distance entre deux individus (ici, entre deux employés).
  2. Calculer la distance entre le nouvel individu et tous les individus connus.
  3. Garder les k plus proches (les k distances les plus faibles).
  4. Renvoyer la moyenne des valeurs cibles de ces k voisins (ici, leurs salaires).

La distance utilisée dans le programme

La fonction distance du programme calcule une distance entre deux employés en tenant compte de trois critères : le sexe (converti en ±1), l'expérience et les études. 

def sexe_vers_entier(e):
    if e['sexe'] == 'F':
        return 1
    else:
        return -1

def distance(e1, e2):
    s = (sexe_vers_entier(e1) - sexe_vers_entier(e2))**2   # différence sexe
    s = s + (e1['experience'] - e2['experience'])**2        # différence expérience
    s = s + (e1['etudes'] - e2['etudes'])**2                # différence études
    return sqrt(s)
Rappel — Distance euclidienne

La formule \(d = \sqrt{(x_1-x_2)^2 + (y_1-y_2)^2 + \ldots}\) est la généralisation en plusieurs dimensions du théorème de Pythagore. Ici on l'applique à trois « coordonnées » : sexe, expérience, études.

Test sur deux profils

Applique salaire_par_proximite aux deux profils suivants sur le jeu de test : 

profil_f = {'experience': 3, 'etudes': 3, 'sexe': 'F'}
profil_m = {'experience': 3, 'etudes': 3, 'sexe': 'M'}
Calcul manuel des distances – profil_f

Pour profil_f ('F', expérience 3, études 3), calculons les distances aux 6 employés de test :

EmployésexeexpetuDistance
1 – F, 5, 3, 2400€F→153 \(\sqrt{0^2+2^2+0^2}=2\)
2 – M, 3, 3, 2550€M→-133 \(\sqrt{2^2+0^2+0^2}=2\)
3 – F, 5, 5, 2500€F→155 \(\sqrt{0^2+2^2+2^2}\approx2{,}83\)
4 – M, 3, 5, 2800€M→-135 \(\sqrt{2^2+0^2+2^2}\approx2{,}83\)
5 – F, 2, 5, 2300€F→125 \(\sqrt{0^2+1^2+2^2}\approx2{,}24\)
6 – M, 2, 3, 2700€M→-123 \(\sqrt{2^2+1^2+0^2}\approx2{,}24\)

Les 3 plus proches (distance 2) : employé 1 (F, 2400€), employé 2 (M, 2550€), puis à distance ≈ 2,24 l'employé 5 (F, 2300€) ou 6 (M, 2700€) — selon le tri, le résultat change. En pratique, le programme renvoie une moyenne autour de 2 417 €.

Pour profil_m, le calcul est symétrique : les employés proches des hommes sont d'abord les hommes avec les mêmes caractéristiques. Le programme renvoie environ 2 683 €.

Le problème — Le sexe dans la distance crée un biais

Deux profils identiques en termes d'expérience et d'études obtiennent des salaires proposés différents (~2 417 € vs ~2 683 €) — uniquement parce que leur sexe influence les voisins sélectionnés !

En incluant le sexe dans le calcul de la distance, on fait en sorte que les voisins des femmes sont principalement des femmes (qui ont historiquement des salaires plus bas), et vice versa. L'algorithme reproduit et perpétue le biais salarial qu'on cherche pourtant à mesurer.

Correction

Pour obtenir une recommandation équitable, il faut retirer le sexe du calcul de la distance : deux personnes avec la même expérience et le même niveau d'études sont également proches, quel que soit leur sexe.

Indice

Il suffit de supprimer la ligne qui ajoute la contribution du sexe dans distance, ou d'écrire une nouvelle version distance_sans_sexe : 

def distance_sans_sexe(e1, e2):
    s = (e1['experience'] - e2['experience'])**2
    s = s + (e1['etudes'] - e2['etudes'])**2
    return sqrt(s)
Éditeur Python — Teste et corrige le biais
Correction — Question 4

Résumé : la source du biais est la présence du sexe dans le calcul de la distance. En l'excluant, les deux profils identiques (experience 3, études 3) obtiennent le même salaire proposé : 2 550 €.

Avant correction : profil_f → ~2 417 €, profil_m → ~2 683 €
Après correction : profil_f → 2 550 €, profil_m → 2 550 €

Ce cas illustre concrètement comment un algorithme peut reproduire des inégalités existantes s'il est mal conçu — et comment une correction technique simple peut suffire à le rendre équitable.

Pièges fréquents & astuces de vérification

Piège 1 — Variable vs chaîne de caractères

C'est exactement le bug de calcul_ecart_sexe : écrire 'employes' (avec guillemets) au lieu de employes (sans guillemets). La chaîne 'employes' est itérable en Python — elle donne les caractères 'e', 'm', 'p', … — ce qui ne lève pas d'erreur immédiatement mais produit un résultat absurde.

Réflexe : si ta fonction renvoie None de façon inattendue, vérifie que tu n'as pas mis de guillemets autour d'un nom de variable.

Piège 2 — Oublier le cas « aucun employé »

Toute fonction qui calcule une moyenne doit gérer le cas où la liste est vide. Diviser par zéro en Python lève une ZeroDivisionError. Toujours tester : assert salaire_moyen_condition([], 'sexe', 'F') == None.

Piège 3 — Différence vs pourcentage

L'écart en euros (moy_h - moy_f) et l'écart en pourcentage ((moy_h - moy_f) / moy_h * 100) sont deux choses très différentes. Sur le jeu de test, l'un vaut ~283 (euros), l'autre ~10,6 (pourcent). L'énoncé demande toujours le pourcentage.

Piège 4 — Biais dans la distance k-NN

Intégrer une caractéristique sensible (sexe, origine…) dans le calcul de distance revient à regrouper les individus par cette caractéristique — et donc à reproduire les inégalités présentes dans les données d'entraînement. C'est un problème éthique majeur en apprentissage automatique.

Piège 5 — Confondre is None et == None

En Python, pour tester si une variable vaut None, la bonne pratique est d'écrire if x is None (et non if x == None). Les deux fonctionnent pour None, mais is None est plus idiomatique et plus sûr avec des objets personnalisés.

Récapitulatif – Ce que tu dois retenir
  • Une liste de dictionnaires se parcourt avec une boucle for emp in employes — on accède aux champs avec emp['champ'].
  • La moyenne conditionnelle : filtrer d'abord, vérifier que la liste n'est pas vide, puis diviser.
  • 'employes' ≠ employes : les guillemets transforment une variable en chaîne de caractères.
  • L'écart en pourcentage se calcule avec \(\frac{\text{moy}_H - \text{moy}_F}{\text{moy}_H} \times 100\), pas une simple soustraction.
  • Dans un algorithme k-NN, inclure une caractéristique sensible dans la distance reproduit les biais des données d'entraînement.
  • Toujours écrire des fonctions de test avec au moins 3 assertions pour détecter les bugs rapidement.
