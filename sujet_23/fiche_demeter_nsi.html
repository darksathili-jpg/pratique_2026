1 — Rappels : binaire, BCD et parité

Conversion binaire → décimal

En Python, int(chaine, 2) convertit une chaîne de caractères représentant un nombre en binaire vers sa valeur décimale.

Exemples 
int("00101010", 2)    # → 42
int("010010001100", 2) # → 1164
int("1010", 2)         # → 10
int("0110", 2)         # → 6

Pour accéder à une portion d'une chaîne, on utilise le slicing : trame[a:b] renvoie les caractères d'indice a inclus jusqu'à b exclu. 

trame = "0010101011001000010010001100011000101101"
trame[0:8]   # → "00101010"   (8 premiers bits = ID)
trame[16:28] # → "010010001100" (12 bits = température)
trame[28:36] # → "01100010"   (8 bits = humidité)

Le codage BCD (Décimal Codé Binaire)

En BCD, chaque chiffre décimal est codé indépendamment sur 4 bits. Pour coder « 62 », on code « 6 » puis « 2 » séparément :

Exemple — Décoder 01100010 en BCD
GroupeBitsDécimal
Dizaines01106
Unités00102
Résultat62 %
 
hum_bits = "01100010"
d1 = int(hum_bits[0:4], 2)   # → 6  (dizaines)
d2 = int(hum_bits[4:8], 2)   # → 2  (unités)
humidite = d1 * 10 + d2       # → 62

Cas particulier : 100 % est codé 10100000 (ce qui donnerait 10 et 0, mais 10 n'est pas un chiffre BCD valide). On traite ce cas séparément : if hum_bits == "10100000": return 100.

La parité — détecter les erreurs de transmission

Imagine que tu transmets un message par radio et qu'un parasite flip un bit en chemin. Comment le destinataire peut-il s'en apercevoir ? En ajoutant un bit de parité après chaque bloc de données.

Définition — Bit de parité impaire

Le bit de parité d'un bloc vaut 1 si le nombre de bits à 1 dans ce bloc est impair, et 0 si ce nombre est pair. Ainsi, le nombre total de 1 (bloc + bit de parité) est toujours pair. 

def parite(bloc):
    """Renvoie le bit de parité d'un bloc binaire."""
    return 1 if bloc.count('1') % 2 == 1 else 0

parite("00101010")   # 3 uns → impair → 1
parite("11001000")   # 3 uns → impair → 1
parite("010010001100") # 4 uns → pair → 0

2 — La trame DEMETER

Chaque module DEMETER envoie une trame de exactement 40 bits. La trame est découpée en 5 blocs contigus :

ID
[0:8] 00101010 = 42
Clé sécurité
[8:16] 11001000 = 200
Température
[16:28] 010010001100 26,4 °C
Humidité
[28:36] 01100010 62 %
Contrôle
[36:40] 1101 parité
ChampIndicesTailleEncodageFormule
ID[0:8]8 bits Binaire classique int(trame[0:8], 2)
Clé de sécurité[8:16]8 bits Non décodée dans ce sujet—
Température[16:28]12 bits Binaire + offset (int(trame[16:28], 2) − 900) / 10
Humidité[28:36]8 bits BCD (4 bits + 4 bits) d1×10 + d2
Contrôle[36:40]4 bits Parité des 4 blocs 1 si nb de 1 impair, 0 sinon
Vérification pas à pas — température

Bits température : 010010001100
Valeur binaire → décimal : \(0 \times 2^{11} + 1 \times 2^{10} + 0 \times 2^9 + \ldots = 1164\)
Température en °C : \(\dfrac{1164 - 900}{10} = \dfrac{264}{10} = 26{,}4 \text{ °C}\)

La valeur 900 est un décalage (offset) qui permet de représenter des températures négatives. La valeur minimale codée est \(\dfrac{0 - 900}{10} = -90 \text{ °C}\), la maximale est \(\dfrac{4095 - 900}{10} = 319{,}5 \text{ °C}\).

La classe Transmission

Architecture de la classe 
class Transmission:
    def __init__(self, trame):
        self._id          = None
        self._temperature = None
        self._humidite    = None
        self._trame       = trame
        self.decoder()        # décode tout à la construction

    def decoder(self):
        self.decoder_id()          # bits [0:8]
        self.decoder_temperature() # bits [16:28] — À COMPLÉTER
        self.decoder_humidite()    # bits [28:36] — À COMPLÉTER

    def decoder_id(self):
        self._id = int(self._trame[0:8], 2)

    def est_valide(self):
        return False  # À COMPLÉTER

Question 1 — Décoder la température et l'humidité

Énoncé — Question 1

Q1 Écrire les méthodes decoder_temperature(self) et decoder_humidite(self) de la classe Transmission.

  • decoder_temperature doit stocker dans self._temperature la température en °C sous forme de flottant.
  • decoder_humidite doit stocker dans self._humidite l'humidité en pourcentage sous forme d'entier, en tenant compte du cas spécial 100 %.

Vérifier avec la trame d'exemple que les résultats sont bien 26,4 °C et 62 %.

Indice 1 — Température

Les bits de température sont self._trame[16:28] (12 bits). Convertir en entier avec int(..., 2), soustraire 900, diviser par 10. Le résultat est un float.

Indice 2 — Humidité BCD 
hum_bits = self._trame[28:36]
if hum_bits == "10100000":        # cas spécial : 100 %
    self._humidite = 100
else:
    d1 = int(hum_bits[0:4], 2)   # dizaines
    d2 = int(hum_bits[4:8], 2)   # unités

Correction — Question 1

Question 2 — Valider une trame par parité

Énoncé — Question 2

Q2 Écrire la méthode est_valide(self) qui renvoie True si les 4 bits de contrôle trame[36:40] correspondent bien aux bits de parité des 4 blocs de données, False sinon.

Rappel — Les 4 blocs et leurs bits de contrôle

Bloc 1 → ID [0:8] → bit de contrôle [36]
Bloc 2 → Clé [8:16] → bit de contrôle [37]
Bloc 3 → Température [16:28] → bit de contrôle [38]
Bloc 4 → Humidité [28:36] → bit de contrôle [39]

Un bit de parité vaut 1 si le bloc contient un nombre impair de '1', 0 sinon.

Indice 1 
def parite(bloc):
    return '1' if bloc.count('1') % 2 == 1 else '0'

Attention : le bit de contrôle dans la trame est un caractère ('0' ou '1'), pas un entier. Comparer des caractères entre eux.

Indice 2 
blocs = [
    self._trame[0:8],    # ID
    self._trame[8:16],   # Clé
    self._trame[16:28],  # Température
    self._trame[28:36],  # Humidité
]
for i, bloc in enumerate(blocs):
    bit_attendu = '1' if bloc.count('1') % 2 == 1 else '0'
    if self._trame[36 + i] != bit_attendu:
        return False
return True
Éditeur Python — Complète est_valide
Correction — Question 2

Point de vigilance : self._trame[36 + i] renvoie un caractère ('0' ou '1'). Il faut comparer avec une chaîne, pas un entier. C'est aussi pour cette raison que la version non robuste plantera sur une trame de 38 bits à i = 2 : trame[38] lèvera une IndexError.

La ligne from Transmission import Transmission utilise une majuscule. Or le fichier s'appelle transmission.py (minuscule).

Question 4 — Rendre la classe robuste

Énoncé — Question 4

Q4 Proposer des corrections de la classe Transmission et du fichier analyse.py permettant de les rendre robustes face aux anomalies identifiées en Q3, afin de pouvoir effectuer l'analyse des 308 trames valides.

Stratégie — Deux niveaux de robustesse

Correction 1 — immédiate : corriger l'import dans analyse.py.

Correction 2 — défensive : dans Transmission, vérifier len(self._trame) == 40 dès le constructeur. Si ce n'est pas le cas, on ne décode rien et est_valide() renvoie False. Les méthodes get_* renverront alors None, ce qui est filtré par le if t.est_valide() dans analyse.py.

Cette approche est préférable à un try/except qui masquerait des erreurs de programmation réelles.

Indice 1

Le plus tôt possible : dans __init__, avant d'appeler self.decoder(). Si la longueur est incorrecte, on n'appelle pas decoder() du tout — les attributs restent à None. 

def __init__(self, trame):
    self._id = self._temperature = self._humidite = None
    self._trame = trame
    if len(trame) == 40:      # ← garde
        self.decoder()        # décode seulement si la trame est complète
Indice 2

Si la trame n'a pas 40 bits, est_valide doit renvoyer False sans même tenter d'accéder aux bits de contrôle. Ajouter un garde en début de méthode : 

def est_valide(self):
    if len(self._trame) != 40:
        return False          # trame corrompue → rejetée
    # ... vérification de parité habituelle
Éditeur Python — Version robuste complète
Correction — Question 4 (classe + analyse.py)

Correction 1 — analyse.py ligne 2 :

# AVANT (bugué sur Linux/macOS) :
from Transmission import Transmission

# APRÈS (correct) :
from transmission import Transmission   # 't' minuscule

Correction 2 — transmission.py :

Pourquoi ne pas utiliser try/except ? Le try/except masque toutes les exceptions — y compris les erreurs de logique que tu n'as pas anticipées. La vérification explicite de longueur est plus précise et plus lisible : elle dit exactement pourquoi la trame est rejetée.

Résultat attendu sur data.txt : 308 températures valides sur 314 trames (3 de longueur incorrecte + 3 de parité incorrecte rejetées).

Pièges fréquents & astuces de vérification

Piège 1 — Comparer un caractère avec un entier

self._trame[36] renvoie le caractère '1' ou '0', pas l'entier 1 ou 0. Il faut donc comparer avec des chaînes : if self._trame[36 + i] != bit_attendu: où bit_attendu est '1' ou '0' (avec guillemets). Comparer avec != 1 (entier) donnera toujours True et la trame sera toujours invalide.

Piège 2 — Oublier le cas spécial 100 % en BCD

Le code 10100000 pour 100 % n'est pas valide en BCD standard (le groupe 1010 vaut 10 en décimal, ce qui n'est pas un chiffre décimal). Sans traitement particulier, on obtiendrait 100 par calcul — mais par hasard. Il est préférable de le traiter explicitement pour la lisibilité et la robustesse, et parce que la spécification de l'énoncé l'impose.

Piège 3 — Tester la validité avant d'utiliser get_temperature()

Pour une trame corrompue (longueur incorrecte), get_temperature() renvoie None. Si tu construis une liste de températures sans filtrer par est_valide(), tu obtiendras des None dans la liste — ce qui fera planter matplotlib lors du tracé. Toujours filtrer : [t.get_temperature() for t in transmissions if t.est_valide()].

Piège 4 — La casse des imports Python

from Transmission import Transmission et from transmission import Transmission sont équivalents sur Windows, mais différents sur Linux et macOS. Toujours faire correspondre exactement la casse du nom du fichier. C'est une source fréquente de bugs qui n'apparaissent qu'en production (souvent sur serveur Linux) alors que tout fonctionnait en développement (sur Windows).

Récapitulatif — Ce que tu dois retenir
  • int(chaine, 2) convertit du binaire vers décimal — indispensable pour décoder les trames.
  • Le slicing trame[a:b] extrait les bits d'indice a à b-1.
  • En BCD, chaque chiffre décimal est codé sur 4 bits indépendamment.
  • Le bit de parité est 1 si le bloc contient un nombre impair de 1, 0 sinon.
  • Toujours vérifier len(trame) == 40 avant d'accéder aux indices — sinon IndexError.
  • Sur Linux/macOS, la casse des noms de modules Python est signifiante.
  • Le filtrage if t.est_valide() dans la compréhension de liste élimine proprement les données corrompues.
