1 — Introduction & Rappels de cours

Le cycle menstruel en quatre actes

Imagine une horloge biologique qui se remet à zéro toutes les 28 jours (en moyenne). Ce cycle se décompose en quatre étapes que l'on numérote de 1 à 4 :

Règles Jours 1–5
Phase folliculaire Jours 6–13
Ovul. J. 14
Phase lutéale Jours 15–28
Phase Numéro Jours du cycle En bref…
Règles 1 1 à 5 Écoulement menstruel
Phase folliculaire 2 6 à 13 Développement d'un follicule dans l'ovaire
Ovulation 3 14 Libération de l'ovocyte
Phase lutéale 4 15 à 28 Développement de la muqueuse utérine

Dans ce sujet, on va modéliser ce cycle en Python : représenter les dates, calculer dans quelle phase se trouve un jour donné, et même générer un fichier d'agenda automatiquement.

Rappel — Les tuples en Python

Un tuple, c'est comme une liste, mais on ne peut pas le modifier après sa création. On le crée avec des parenthèses : 

date = (7, 9, 2025)   # le 7 septembre 2025

Pour accéder à chaque élément, on utilise un indice — attention, ça commence à 0 : 

jour, mois, annee = date   # déballage (unpacking)
print(jour)    # affiche 7
print(mois)    # affiche 9
print(annee)   # affiche 2025
Définition — Tuple

Un tuple est une séquence immuable d'éléments. Dans ce sujet, on représente toujours une date par le tuple (jour, mois, annee), dans cet ordre précis. Par exemple, (3, 7, 2026) désigne le 3 juillet 2026.

Rappel — Les années bissextiles

Une année est bissextile si elle compte 366 jours au lieu de 365. La règle se décompose en trois conditions que tu peux vérifier dans cet ordre :

Règle des années bissextiles

Une année \(a\) est bissextile si et seulement si :

  • Elle est divisible par 4, et
  • Elle n'est pas divisible par 100, ou bien
  • Elle est divisible par 400.

En Python, la divisibilité se teste avec l'opérateur % (modulo) : a % 4 == 0 est True si et seulement si a est divisible par 4.

Trois exemples du sujet
  • 2024 : divisible par 4 ✔, pas par 100 ✔ → bissextile
  • 2100 : divisible par 4 ✔, divisible par 100 ✔, mais pas par 400 ✗ → non bissextile
  • 2000 : divisible par 4 ✔, par 100 ✔, et par 400 ✔ → bissextile

Question 1 — La fonction est_bissextile

Énoncé — Question 1

Q1 Écrire en Python une fonction nommée est_bissextile qui prend en paramètre un entier correspondant à une année et qui renvoie un booléen (True ou False) indiquant si elle est bissextile, en appliquant la règle donnée ci-dessus.

✋ Appel professeur — présente ta réponse ou sollicite de l'aide.

Comment aborder cette question ?

Tu dois traduire la règle en trois conditions Python. La clé, c'est de bien choisir l'ordre des tests. Voici une démarche possible :

  1. Est-ce que l'année est divisible par 4 ? Si non, pas bissextile.
  2. Si oui, est-elle divisible par 100 ? Si non, elle est bissextile.
  3. Si oui, est-elle divisible par 400 ? Si oui, bissextile. Sinon, non.
Indice

La structure générale avec des conditions imbriquées :

if annee % 4 != 0:
    return False
elif annee % 100 != 0:
    return True
elif annee % 400 == 0:
    return True
else:
    return False

Tu peux aussi écrire tout cela en une seule ligne avec des opérateurs logiques (and, or, not). C'est plus élégant mais peut être moins lisible.

Essaie dans l'éditeur ci-dessous :

Correction — Q1 
def est_bissextile(annee):
    if annee % 4 != 0:
        return False
    elif annee % 100 != 0:
        return True
    elif annee % 400 == 0:
        return True
    else:
        return False

def test_est_bissextile():
    assert est_bissextile(2024) == True
    assert est_bissextile(2100) == False
    assert est_bissextile(2000) == True
    assert est_bissextile(2025) == False
    print("Les tests sont validés.")

Version condensée possible (avancée) :
return (annee % 4 == 0 and annee % 100 != 0) or (annee % 400 == 0)

Question 2 — La fonction determiner_phase

Énoncé — Question 2

Q2 Écrire en Python une fonction nommée determiner_phase qui prend en paramètre un entier compris entre 1 et 28 inclus (le jour du cycle) et qui renvoie un entier correspondant au numéro de la phase associée (1, 2, 3 ou 4).

À l'aide d'une assertion, on garantira que l'entier donné en argument est bien compris entre 1 et 28 inclus.
✋ Appel professeur — présente ta réponse ou sollicite de l'aide.

Qu'est-ce qu'une assertion ?

Une assertion, c'est un garde-fou : on dit à Python « je garantis que cette condition est vraie — si elle ne l'est pas, arrête tout et affiche une erreur ». La syntaxe est simple : 

assert condition, "Message d'erreur si la condition est fausse"
Exemple 
assert 1 <= jour <= 28, "Le jour doit être entre 1 et 28"

Si on appelle la fonction avec jour = 0 ou jour = 29, Python lèvera une AssertionError avec le message.

Les conditions de phase

Reporte-toi au tableau de la section 1 : règles si le jour est entre 1 et 5, phase folliculaire entre 6 et 13, ovulation le jour 14, phase lutéale entre 15 et 28. Tu vas enchaîner des conditions if … elif … elif … else.

Indice 
def determiner_phase(jour):
    assert 1 <= jour <= 28, "Jour invalide"
    if 1 <= jour <= 5:
        return 1
    elif 6 <= jour <= 13:
        return ...   # à compléter
    elif jour == 14:
        return ...
    else:
        return ...

Question 3 — Tests de ajouter_jours

Énoncé — Question 3

Q3 La fonction ajouter_jours est déjà fournie dans cycle_menstruel.py. Elle prend une date (tuple (jour, mois, annee)) et un nombre de jours à ajouter, et renvoie la nouvelle date calculée.

Compléter la fonction test_ajouter_jours en ajoutant au moins trois autres tests pertinents. Pour chaque test ajouté, explique brièvement pourquoi ce cas est important à vérifier.

La fonction fournie

Voici le code complet de ajouter_jours que tu dois comprendre avant d'écrire des tests : 

import calendar

def jours_dans_mois(annee, mois):
    if mois == 2:
        return 29 if calendar.isleap(annee) else 28
    elif mois in [1, 3, 5, 7, 8, 10, 12]:
        return 31
    else:
        return 30

def ajouter_jours(date, nb_jours):
    jour, mois, annee = date
    jour = jour + nb_jours
    while jour > jours_dans_mois(annee, mois):
        jour = jour - jours_dans_mois(annee, mois)
        mois = mois + 1
        if mois > 12:    # passage à l'année suivante
            mois = 1
            annee = annee + 1
    return (jour, mois, annee)
Test déjà fourni 
def test_ajouter_jours():
    assert ajouter_jours((7, 9, 2025), 3) == (10, 9, 2025)
    # → cas simple : reste dans le même mois

Quels cas faut-il tester ?

Un bon jeu de tests couvre les cas limites, pas seulement les cas normaux. Pose-toi cette question : « Où est-ce que ça pourrait mal tourner ? »

Idées de cas limites
  • Passage au mois suivant : ajouter des jours qui dépassent la fin du mois.
  • Passage à l'année suivante : ajouter des jours en décembre qui débordent sur janvier.
  • Février en année bissextile : le 29 février existe seulement certaines années.
  • Ajout de 0 jours : la date ne doit pas changer.
  • Ajout de 28 jours : c'est exactement la durée du cycle menstruel !
Indice — valeurs pré-calculées 
# Passage au mois suivant (30 sept + 5 = 5 oct)
ajouter_jours((30, 9, 2025), 5)  →  (5, 10, 2025)

# Passage à l'année suivante (31 déc + 1 = 1 jan)
ajouter_jours((31, 12, 2025), 1) →  (1, 1, 2026)

# Février bissextile 2024 (28 fév + 1 = 29 fév)
ajouter_jours((28, 2, 2024), 1)  →  (29, 2, 2024)

# Février non bissextile 2025 (28 fév + 1 = 1 mars)
ajouter_jours((28, 2, 2025), 1)  →  (1, 3, 2025)

# Ajout de 28 jours à une date de début de règles
ajouter_jours((12, 3, 2026), 28) →  (9, 4, 2026)

def test_ajouter_jours():
    # Test 1 (fourni) — cas simple dans le même mois
    assert ajouter_jours((7, 9, 2025), 3) == (10, 9, 2025)

    # Test 2 — passage au mois suivant
    # Vérifie que la boucle while franchit correctement la fin d'un mois de 30 jours.
    assert ajouter_jours((30, 9, 2025), 5) == (5, 10, 2025)

    # Test 3 — passage à l'année suivante
    # Le cas mois > 12 réinitialise à janvier de l'année suivante.
    assert ajouter_jours((31, 12, 2025), 1) == (1, 1, 2026)

    # 2024 est bissextile : le 29 février existe.
    assert ajouter_jours((28, 2, 2024), 1) == (29, 2, 2024)

    # Test 5 — février non bissextile
    # 2025 n'est pas bissextile : après le 28 vient directement le 1er mars.
    assert ajouter_jours((28, 2, 2025), 1) == (1, 3, 2025)

Question 4 — Corriger le format iCalendar

Énoncé — Question 4

Q4 La fonction calendrier_cycles (déjà fournie) génère un calendrier iCalendar des prochaines dates de règles dans les 100 jours suivant une date de départ. En l'observant avec test_calendrier_cycles, on constate que le calendrier produit n'est pas dans un format valide.

Identifier le problème, proposer une démarche de résolution et la mettre en œuvre. ✋ Appel professeur — présente ta réponse ou sollicite de l'aide.

Le format iCalendar — qu'est-ce que c'est ?

Un fichier iCalendar (.ics) est un format texte que tous les agendas numériques comprennent (Google Calendar, Apple Calendrier, Outlook…). Chaque événement s'écrit ainsi : 

BEGIN:VEVENT
SUMMARY:Règles
DTSTART:20260703
END:VEVENT

La valeur de DTSTART suit le format AAAAMMJJ, soit toujours 8 caractères. Le 3 juillet 2026 s'écrit donc 20260703 et non 202673.

Où est le bug ?

Voici la ligne problématique dans calendrier_cycles : 

date = str(annee) + str(mois) + str(jour)

Avec annee = 2026, mois = 7, jour = 3, cette concaténation donne : 

"2026" + "7" + "3"  =  "202673"   # ← 6 caractères, format invalide !
Ce qui est attendu

"2026" + "07" + "03" = "20260703"  — toujours 8 caractères.
Les mois et jours inférieurs à 10 doivent être complétés par un zéro à gauche.

Deux solutions possibles

Solution A — méthode zfill

str(n).zfill(k) complète la chaîne avec des zéros à gauche pour atteindre une longueur de k. 

date = str(annee) + str(mois).zfill(2) + str(jour).zfill(2)
# "2026" + "07" + "03" = "20260703" ✔
Solution B — f-string avec formatage :02d

Avec une f-string, {n:02d} formate l'entier sur au moins 2 chiffres, avec un zéro si nécessaire. 

date = f"{annee}{mois:02d}{jour:02d}"
# "20260703" ✔

Essaie dans l'éditeur — corrige la fonction :

Correction — Q4

Problème identifié : la ligne

date = str(annee) + str(mois) + str(jour)

ne complète pas les mois et jours inférieurs à 10 avec un zéro. Pour le 3 juillet 2026, elle produit "202673" (6 caractères) au lieu de "20260703" (8 caractères requis).

Correction (avec f-string) :

date = f"{annee}{mois:02d}{jour:02d}"

Correction (avec zfill) :

date = str(annee) + str(mois).zfill(2) + str(jour).zfill(2)

Les deux solutions sont équivalentes. La f-string est généralement considérée plus lisible en Python moderne.

Pièges fréquents & points de vigilance

Piège 1 — Confondre l'indice du tuple et sa valeur

Le tuple (7, 9, 2025) représente le 7 septembre 2025. L'accès date[0] renvoie 7 (le jour), pas le mois. Si tu mélanges les indices, tu te retrouves à calculer des mois comme des jours. Utilise toujours le déballage (unpacking) : jour, mois, annee = date.

Piège 2 — Le cycle commence au jour 1, pas au jour 0

L'assertion assert 1 <= jour <= 28 est correcte. Écrire assert 0 <= jour <= 28 ou assert 0 <= jour < 28 est une erreur : le jour 0 n'existe pas dans un cycle, et le jour 28 est la phase lutéale — il doit être inclus.

Piège 3 — Oublier str() avant de concaténer

str(annee) + str(mois) + str(jour) — chaque composante doit être convertie en chaîne. Si tu écris str(annee) + mois + jour, Python lève une TypeError car on ne peut pas additionner une chaîne et un entier.

Piège 4 — Ne pas compléter avec des zéros (zfill ou :02d)

C'est le cœur de la Question 4 : str(7) renvoie "7", pas "07". Le format iCalendar exige exactement 8 caractères pour DTSTART. Utilise str(mois).zfill(2) ou f"{mois:02d}" pour forcer deux chiffres.

Astuces de vérification rapide
  • Longueur de la date : assert len(date_str) == 8 doit toujours passer quel que soit le mois ou le jour.
  • Aller-retour sur les dates : si tu ajoutes 0 jours à une date, tu dois retrouver exactement la même date.
  • AssertionError volontaire : pour tester ton assertion dans determiner_phase, essaie d'appeler determiner_phase(0) ou determiner_phase(29) — Python doit lever une erreur.

Mini-exercices d'extension

Extension A — Cycle de 30 jours

Le sujet suppose un cycle régulier de 28 jours. Dans la réalité, les cycles varient entre 21 et 35 jours.

Modifie la fonction calendrier_cycles pour qu'elle accepte un second paramètre facultatif duree_cycle = 28. Le calendrier doit alors utiliser cette durée au lieu de 28.

Vérifie qu'avec duree_cycle = 30, on obtient au plus 3 occurrences en 100 jours.

def calendrier_cycles(date_regles, duree_cycle=28):
    cal_lignes = ['BEGIN:VCALENDAR', 'VERSION:2.0', 'PRODID:']
    date_courante = date_regles
    jours_ecoules = 0
    while jours_ecoules + duree_cycle <= 100:
        jour, mois, annee = date_courante
        cal_lignes.append('BEGIN:VEVENT')
        cal_lignes.append('SUMMARY:Règles')
        date = f"{annee}{mois:02d}{jour:02d}"   # format corrigé
        cal_lignes.append('DTSTART:' + date)
        cal_lignes.append('END:VEVENT')
        date_courante = ajouter_jours(date_courante, duree_cycle)
        jours_ecoules += duree_cycle
    cal_lignes.append('END:VCALENDAR')
    return '\n'.join(cal_lignes)

# Avec 30 jours : 3 occurrences (0, 30, 60 jours écoulés — 90+30 > 100 donc on s'arrête)
cal = calendrier_cycles((12, 3, 2026), duree_cycle=30)
occurrences = [l for l in cal.split('\n') if l.startswith('DTSTART')]
assert len(occurrences) == 3
print("OK — 3 occurrences pour un cycle de 30 jours")
Extension B — Assertion sur le mois

La fonction jours_dans_mois(annee, mois) suppose que le mois est entre 1 et 12. Si on lui passe mois = 0 ou mois = 13, elle peut donner un résultat incorrect sans avertir.

Ajoute une assertion au début de jours_dans_mois pour garantir que le mois est bien entre 1 et 12. Écris ensuite un test qui vérifie que cette assertion est bien déclenchée. 

def jours_dans_mois(annee, mois):
    assert 1 <= mois <= 12, f"Mois invalide : {mois} (doit être entre 1 et 12)"
    if mois == 2:
        return 29 if calendar.isleap(annee) else 28
    elif mois in [1, 3, 5, 7, 8, 10, 12]:
        return 31
    else:
        return 30

# Test de l'assertion (doit lever une AssertionError)
try:
    jours_dans_mois(2026, 13)
    print("ERREUR : l'assertion n'a pas été déclenchée !")
except AssertionError:
    print("Correct : AssertionError bien levée pour mois = 13")
Ce que tu dois retenir
  • Un tuple (jour, mois, annee) représente une date — l'ordre compte, les indices commencent à 0.
  • Une assertion protège ta fonction contre des entrées invalides : assert condition, "message".
  • Les années bissextiles suivent la règle : divisible par 4, sauf les multiples de 100, sauf les multiples de 400.
  • Le format iCalendar DTSTART exige exactement 8 caractères : AAAAMMJJ.
  • Pour forcer 2 chiffres : f"{mois:02d}" ou str(mois).zfill(2).
  • Teste toujours les cas limites : changement de mois, changement d'année, février bissextile.
