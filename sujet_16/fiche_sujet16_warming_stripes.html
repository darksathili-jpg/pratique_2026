1 — Rappels essentiels

Rechercher dans une liste de dictionnaires

Dans ce sujet, chaque donnée est un dictionnaire avec deux clés : 'année' (entier) et 'écart' (flottant). Pour trouver la valeur associée à une année précise, on parcourt la liste et on teste chaque dictionnaire : 

donnees = [{"année": 2020, "écart": 1.01}, {"année": 2021, "écart": 0.99}]

def chercher(donnees, annee_cible):
    for dico in donnees:
        if dico["année"] == annee_cible:
            return dico["écart"]   # trouvé !
    return None                    # pas dans la liste
La valeur sentinelle None

Quand une fonction peut ne pas trouver ce qu'elle cherche, la convention Python est de retourner None. C'est une valeur spéciale qui signifie « rien ». On la teste avec if resultat is None: ou if resultat is not None:.

Construire une liste par compréhension

# Extraire toutes les années d'une liste de dicos
annees = [dico["année"] for dico in donnees]

# Extraire toutes les valeurs positives uniquement
positives = [dico["écart"] for dico in donnees if dico["écart"] > 0]

La régression linéaire — intuition sans maths

La fonction prevision trouve la droite qui « colle le mieux » aux n dernières années de données, puis lit sur cette droite la valeur prévue pour une année future. Tu n'as pas à réécrire cette fonction — seulement à corriger moyenne_ecarts dont elle dépend.

Définition – écart de température (anomalie)

L'écart de température (ou anomalie) d'une année est la différence entre la température moyenne mondiale de cette année et la température de référence (moyenne 1901–2000). Un écart positif = plus chaud que la référence (bande rouge). Un écart négatif = plus froid (bande bleue).

4 — Q2 Utiliser derniere_annee_ecart_negatif

Énoncé de la question 2

En utilisant la fonction derniere_annee_ecart_negatif (déjà fournie, qui utilise ta fonction ecart_temperature), déterminer quelle a été la dernière année où la température mondiale était inférieure à la moyenne de référence.

Comprendre derniere_annee_ecart_negatif

Cette fonction est déjà écrite. Voici comment elle fonctionne : elle part de la dernière année du jeu de données (2025), et remonte dans le temps année par année jusqu'à trouver une année avec un écart strictement négatif. 

def derniere_annee_ecart_negatif(datas):
    annee = max([element["année"] for element in datas])  # → 2025
    ecart = ecart_temperature(datas, annee)
    while ecart >= 0:               # tant que l'écart est positif ou nul
        annee = annee - 1           # on recule d'un an
        ecart = ecart_temperature(datas, annee)
    return annee                    # première année avec écart < 0
Résultat attendu

L'appel derniere_annee_ecart_negatif(datas_temperature) renvoie 1977 (écart = −0,03 °C). Cela signifie que depuis 1978, chaque année a eu une température mondiale supérieure ou égale à la moyenne de référence 1901–2000.

5 — Q3 Identifier et corriger le bug de moyenne_ecarts

Énoncé de la question 3

En exécutant prevision(datas, 2040, 20), le résultat obtenu semble absurde compte tenu du réchauffement actuel. Cette absurdité provient d'une erreur de logique dans la fonction moyenne_ecarts. Analyser le code, identifier l'erreur, et la corriger. Que devient alors la prévision pour 2040 ?

Le code bugué fourni

def moyenne_ecarts(annee_debut, annee_fin, datas):
    somme = 0
    compteur = 0
    for dico in datas:
        if annee_debut <= dico["année"] and dico["année"] <= annee_fin:
            somme = somme - dico["écart"]   # ← BUG ICI
            compteur += 1
    return somme / compteur

Analyse du bug

Regarde attentivement la ligne de calcul de la somme : au lieu d'additionner l'écart à la somme accumulée, le code le soustrait.

Bug identifié — soustraction au lieu d'addition

somme = somme - dico["écart"] accumule l'opposé des valeurs. Pour des données récentes toutes positives (ex. 0.88, 0.9, 1.22…), la somme devient très négative, la moyenne aussi, et la régression prédit une tendance à la baisse — absurde au vu du réchauffement.

Effet concret sur les chiffres

VersionMoyenne 2006–2025Prévision 2040
Buguée (-)−0,836 °C (faux !)−0,1054 °C (absurde)
Corrigée (+)+0,836 °C ✓+1,5666 °C ✓
Indice – la correction 
# Remplacer :
somme = somme - dico["écart"]

# Par :
somme = somme + dico["écart"]

Éditeur Python — à toi de jouer !

Correction – Question 3

Ligne buguée :

somme = somme - dico["écart"]   # soustraction → moyenne négative (faux)

Ligne corrigée :

somme = somme + dico["écart"]   # addition → moyenne correcte

Vérification :

print(prevision(datas_temperature, 2040, 20))
# → 1.5666...   (environ +1,57 °C, cohérent avec le réchauffement actuel)

6 — Q4 Compléter graphique — les listes annees et ordonnees

Énoncé de la question 4

On souhaite générer la représentation graphique des warming stripes. Dans la fonction graphique(datas), le code pour gérer les couleurs est déjà préparé. Compléter le code en générant les listes annees et ordonnees.

Indication : pour obtenir des bandes colorées de hauteur uniforme couvrant tout le graphique, la liste ordonnees devra contenir la valeur 1 pour chaque année traitée.

Comprendre la structure de graphique

La fonction utilise ax.bar(annees, ordonnees, ...) pour dessiner des barres verticales. Chaque barre correspond à une année :

Paramètre de bar()RôleValeur attendue
anneesPosition horizontale de chaque barreListe des années : [1851, 1852, …, 2025]
ordonneesHauteur de chaque barre[1, 1, 1, …, 1] — 175 fois
color=cmap(norm(temperatures))Couleur de chaque barreDéjà fourni — bleu → rouge
Pourquoi ordonnees = [1, 1, 1, …] ?

Les warming stripes ne montrent pas la valeur de l'écart sur l'axe vertical — l'information est encodée dans la couleur, pas dans la hauteur. Toutes les barres ont donc la même hauteur (1), ce qui crée l'effet de « bandes » uniformes. Si on mettait les vraies valeurs d'écart, les barres auraient des hauteurs différentes et certaines seraient négatives — ce n'est pas le rendu voulu.

Indice niveau 1 – deux approches 
# Approche 1 : boucle for classique

    annees.append(dico["???"])
    ordonnees.append(???)

# Approche 2 : compréhension de liste

ordonnees = [??? for dico in datas]
Indice niveau 2 – solution 
annees    = [dico["année"] for dico in datas]
ordonnees = [1 for dico in datas]

7 — Pièges fréquents & astuces de vérification

Piège 1 — Retourner 0 au lieu de None pour une année absente (Q1)

Si l'année n'est pas dans la liste, la fonction doit retourner None, pas 0. Retourner 0 serait ambigu : 0 est une vraie valeur dans le CSV (années 1938 et 1939). La fonction derniere_annee_ecart_negatif dépend de ce comportement — si tu retournes 0, elle pensera que l'écart est nul et continuera à remonter dans le temps indéfiniment. 

# ❌ Mauvais
def ecart_temperature(datas, annee):
    for dico in datas:
        if dico["année"] == annee:
            return dico["écart"]
    return 0   # ambigu : 0 est une vraie valeur !

# ✓ Correct
    return None
Piège 2 — Tester None avec == au lieu de is (Q1)

En Python, on teste None avec is None, jamais avec == None. Les deux fonctionnent souvent mais is None est la convention correcte et sera exigée dans les assertions. 

assert ecart_temperature(datas, 1800) is None    # ✓ correct
assert ecart_temperature(datas, 1800) == None    # ✗ fonctionne mais déconseillé
Piège 3 — Confondre les clés du dictionnaire (Q1 et Q4)

Les clés du dictionnaire produit par charger() sont "année" et "écart" — avec accents. Ce n'est pas habituel en Python mais c'est ce que le code fourni crée. Utiliser "annee" sans accent lève une KeyError. 

# ❌ KeyError : pas d'accent → clé inexistante
dico["annee"]   dico["ecart"]

# ✓ Correct : avec accents (comme dans charger())
dico["année"]   dico["écart"]
Piège 4 — Mettre les vraies valeurs d'écart dans ordonnees (Q4)

L'erreur la plus fréquente à la Q4 est d'écrire : ordonnees = [dico["écart"] for dico in datas]. Le graphique afficherait alors des barres de hauteurs différentes (positives et négatives), ce qui n'est pas du tout un warming stripe. L'indication de l'énoncé est claire : ordonnees doit contenir la valeur 1 pour chaque année.

Piège 5 — Ne pas voir le signe dans moyenne_ecarts (Q3)

Le bug est subtil : somme = somme - dico["écart"] au lieu de somme = somme + dico["écart"]. Un seul caractère change tout. Pour le repérer, la stratégie est de calculer à la main la moyenne de 2 ou 3 valeurs connues et de vérifier que le résultat de la fonction correspond.

Astuce — Vérifier la prévision 2040 à la main

Après correction, prevision(datas_temperature, 2040, 20) doit donner environ +1,57 °C. Si tu obtiens une valeur négative ou proche de zéro, le bug dans moyenne_ecarts n'est pas encore corrigé.

Astuce — Test rapide de ecart_temperature

Avant d'appeler derniere_annee_ecart_negatif, vérifie que ta fonction ecart_temperature renvoie bien None pour une année absente. Si elle renvoie 0, la boucle while ecart >= 0 ne s'arrêtera jamais sur une année absente et plantera avec une erreur infinie.
