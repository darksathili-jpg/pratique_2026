Contexte & code fourni

Les cartes mémoire (flashcards) sont des supports de révision comportant une question au recto et la réponse au verso. Ce sujet implémente le système des boîtes de Leitner : chaque carte possède un niveau d'avancement de 0 à 4. Plus le niveau est élevé, plus le délai avant la prochaine révision est long.

Règles du système Leitner

Si l'utilisateur répond correctement, la carte monte d'un niveau (maximum 4).
S'il se trompe, la carte retombe au niveau 0.
Dans les deux cas, date_prochaine est recalculée selon le nouveau niveau.

Code fourni — cartes.py

import datetime

def date_future(nb_jours):
    """Renvoie la date située nb_jours après aujourd'hui."""
    return datetime.date.today() + datetime.timedelta(days=nb_jours)

# Délais en jours pour chaque niveau (index 0 à 4)
DELAIS = [1, 3, 7, 15, 30]

class Carte:
    def __init__(self, question, reponse):
        self.question = question
        self.reponse  = reponse
        self.niveau   = 0
        self.date_prochaine = datetime.date.today()

    def __repr__(self):
        return f"<Carte: {self.question} (Niveau {self.niveau})>"

    # ← méthode traiter_reponse à écrire (Question 1)
Point clé — notation pointée

Les instances de Carte s'interrogent avec la notation pointée : c.niveau, c.date_prochaine, c.question. C'est différent des dictionnaires où on utilise les crochets d['cle'].

Question 1 — Méthode traiter_reponse

Question 1 / 3

Écrire la méthode traiter_reponse(self, succes) de la classe Carte.

  • Si succes vaut True → le niveau augmente de 1 (sans dépasser 4).
  • Si succes vaut False → le niveau retombe à 0.
  • Dans les deux cas, self.date_prochaine est mise à jour avec date_future(DELAIS[self.niveau]) après la modification du niveau.
Indice

Pensez à modifier self.niveau avant de calculer self.date_prochaine.
Pour plafonner à 4 sans if : min(self.niveau + 1, 4).
Ensuite : self.date_prochaine = date_future(DELAIS[self.niveau]).

✔ Correction 
def traiter_reponse(self, succes):
    if succes:
        self.niveau = min(self.niveau + 1, 4)
    else:
        self.niveau = 0
    self.date_prochaine = date_future(DELAIS[self.niveau])

Points clés : on utilise min(..., 4) pour ne pas dépasser le niveau 4, et on recalcule la date après avoir mis à jour le niveau — sinon on appliquerait le délai de l'ancien niveau.

Question 2 — Fonction extraire_cartes_du_jour

Question 2 / 3

Écrire la fonction extraire_cartes_du_jour(paquet, date_jour).
Elle retourne une nouvelle liste contenant uniquement les cartes dont date_prochaine <= date_jour.

Les objets datetime.date sont comparables avec les opérateurs usuels <, <=, ==, etc.

Indice

Une compréhension de liste est idéale ici :
[carte for carte in paquet if carte.date_prochaine <= date_jour]

✔ Correction — deux approches

Approche 1 (compréhension de liste) :

def extraire_cartes_du_jour(paquet, date_jour):
    return [carte for carte in paquet
            if carte.date_prochaine <= date_jour]

Approche 2 (boucle explicite) :

def extraire_cartes_du_jour(paquet, date_jour):
    resultat = []
    for carte in paquet:
        if carte.date_prochaine <= date_jour:
            resultat.append(carte)
    return resultat

Piège classique : utiliser < au lieu de <= — on raterait les cartes dues exactement aujourd'hui.

Question 3 — Débogage de extraire_cartes_a_renforcer

Question 3 / 3

La fonction doit retourner uniquement les cartes dont le niveau est le minimum du paquet. Le code fourni contient une faille logique. Identifiez-la et proposez une correction.

Code défaillant fourni

def extraire_cartes_a_renforcer(paquet):
    if len(paquet) == 0:
        return []

    niveau_min = paquet[0].niveau
    a_renforcer = []

    for carte in paquet:
        if carte.niveau < niveau_min:
            niveau_min = carte.niveau
            a_renforcer.append(carte)    # ← BUG ICI
        elif carte.niveau == niveau_min:
            a_renforcer.append(carte)

    return a_renforcer

Trace d'exécution

Paquet de test : c1.niveau=2, c2.niveau=1, c3.niveau=2
Niveau minimum attendu : 1 → seule c2 doit être retournée.

🐛 Explication du bug

Lorsque if carte.niveau < niveau_min est vrai, la carte est ajoutée sans vider la liste. Les cartes précédemment ajoutées (à l'ancien niveau supérieur, devenu invalide) restent dans a_renforcer. Résultat : c1 (niveau 2) et c2 (niveau 1) sont retournées ensemble, alors que seule c2 devrait l'être.

✔ Correction — deux approches

Approche 1 : réinitialiser la liste quand un nouveau minimum est trouvé

def extraire_cartes_a_renforcer(paquet):
    if len(paquet) == 0:
        return []
    niveau_min = paquet[0].niveau
    a_renforcer = []
    for carte in paquet:
        if carte.niveau < niveau_min:
            niveau_min = carte.niveau
            a_renforcer = [carte]      # ← réinitialisation !
        elif carte.niveau == niveau_min:
            a_renforcer.append(carte)
    return a_renforcer

Approche 2 : deux passes — trouver le min, puis filtrer

def extraire_cartes_a_renforcer(paquet):
    if len(paquet) == 0:
        return []
    niveau_min = min(c.niveau for c in paquet)
    return [c for c in paquet if c.niveau == niveau_min]

L'approche 2 est plus lisible et évite toute ambiguïté. L'approche 1 corrige le code original avec un changement minimal : remplacer .append(carte) par = [carte].

Pièges fréquents & récapitulatif

Piège 1 — Mettre à jour la date avant le niveau

En Q1, si on appelle date_future(DELAIS[self.niveau]) avant de modifier self.niveau, on applique le délai de l'ancien niveau. Il faut toujours modifier le niveau en premier.

Piège 2 — Oublier le plafond à 4

Sans min(..., 4), un niveau 4 pourrait passer à 5 et provoquer une IndexError lors de l'accès à DELAIS[5] (la liste n'a que 5 éléments, d'index 0 à 4).

Piège 3 — Utiliser < au lieu de <= en Q2

La condition correcte est carte.date_prochaine <= date_jour. Avec < strict, les cartes dont la révision est prévue exactement aujourd'hui ne seraient pas sélectionnées.

Piège 4 — Ne pas réinitialiser a_renforcer en Q3

C'est le bug classique de la recherche de minimum avec collecte simultanée. Quand on découvre un nouveau minimum, toutes les cartes du niveau précédent sont désormais invalides. Il faut écrire a_renforcer = [carte] et non a_renforcer.append(carte).

Récapitulatif — Ce que tu dois retenir
  • En POO, toujours mettre à jour l'état (self.niveau) avant de recalculer les propriétés dérivées (self.date_prochaine).
  • Utiliser min(..., 4) pour plafonner sans conditionnelle supplémentaire.
  • Les comparaisons de dates avec <= incluent le jour courant — ne pas utiliser <.
  • Quand on cherche un minimum en collectant des éléments, réinitialiser la liste dès qu'un nouveau minimum est trouvé.
  • L'approche deux passes (trouver le min avec min(), puis filtrer avec une compréhension) est souvent plus claire et moins sujette aux bugs.
