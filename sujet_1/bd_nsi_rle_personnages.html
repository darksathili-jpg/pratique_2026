2 · Principe RLE
RLE = écrire (compte, valeur) pour chaque suite identique.
avant  [4, 4, 4, 0, 5, 5]
34 · 10 · 25
après  [3, 4, 1, 0, 2, 5]

3 · Le piège de la taille
La liste RLE est-elle toujours plus courte ?
[1, 2, 3, 4, 5] # 5 valeurs
→ [1,1, 1,2, 1,3, 1,4, 1,5]
# longueur 10 — doublée !
NON ! Chaque plage k=1 génère 2 éléments.
Règle : n plages → taille 2n
RLE utile ssi 2p ≤ n
Contre-ex : [1,2,3,4] → ×2
A retenir
RLE n'est pas toujours plus court !
4 · Écrire decodage_rle
Lire la liste RLE 2 par 2 : d'abord le compte, puis la valeur.
def decodage_rle(liste_rle):
  res = []
  i = 0
  while i < len(liste_rle):
    k = liste_rle[i]
    v = liste_rle[i+1]
    res.extend([v]*k)
    i += 2 # PAS DE 2 !
  return res
Piège classique
i += 1 → décalage total !
i += 2 → correct
Test
decodage(codage(L)) == L → True

6 · La solution — découper !
Si k > 255, on découpe en morceaux de 255 max !
while k > 255:
  liste_rle.append(255)
  liste_rle.append(c)
  k -= 255
liste_rle.append(k)
liste_rle.append(c)
Bonus : le décodeur n'est pas modifié — il lit les couples normalement !
A retenir
256 noirs → [255,0] + [1,0]
decodage_rle reste identique
