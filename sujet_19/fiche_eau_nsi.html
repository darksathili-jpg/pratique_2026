1 — Rappels indispensables

Chercher un élément dans une liste de dictionnaires

Deux besoins reviennent dans ce sujet : trouver un élément précis par son nom, et agréger des valeurs selon une clé (le district). Les deux reposent sur la même structure de parcours.

Patron 1 — Recherche par nom 
def trouver_par_nom(reservoirs, nom):
    for r in reservoirs:
        if r["nom"] == nom:
            return r      # trouvé → on renvoie le dictionnaire
    return None           # pas trouvé (ne devrait pas arriver selon l'énoncé)
Patron 2 — Agrégation par clé (volume par district) 
resultat = {}                           # dictionnaire de résultats
for r in reservoirs:
    cle = r["district"]                 # clé d'agrégation
    if cle not in resultat:
        resultat[cle] = 0               # initialiser à 0 si absent
    resultat[cle] += r["volume"]        # accumuler

Calculer un taux de remplissage

Formule — Taux de remplissage

$$\text{taux} = \frac{\text{volume}}{\text{capacite}} \times 100$$ Un réservoir est en pénurie si ce taux est strictement inférieur à 20 %. La condition Python est donc : volume / capacite * 100 < 20.

Construire la moyenne — le diviseur correct

La moyenne — erreur classique de diviseur

La moyenne de \(n\) valeurs se calcule en divisant leur somme par \(n\), pas par \(n-1\) (qui est le diviseur de la variance en statistiques). 

# CORRECT
moyenne = somme / len(liste)

# INCORRECT (bug du sujet)
moyenne = somme / (len(liste) - 1)   # divise par n-1 → résultat trop grand
                                      # et ZeroDivisionError si liste de 1 élément

Question 3 — Tester et corriger volume_moyen

Énoncé — Question 3

Q3 La fonction volume_moyen est incorrecte. Proposer des assertions qui mettent le ou les problèmes en évidence, puis proposer une version corrigée.

Le code fourni :

def volume_moyen(reservoirs):
    somme_totale = 0
    for r in reservoirs:
        somme_totale += r["volume"]
    moyenne = somme_totale / (len(reservoirs)-1)   # ← BUG
    return moyenne
Analyse du bug — diviseur n-1 au lieu de n

Le diviseur devrait être len(reservoirs) (le nombre de réservoirs), mais il est len(reservoirs) - 1.

CasRésultat buguéRésultat attenduProblème
1 réservoir (vol. 42 000) ZeroDivisionError 42 000 Division par zéro !
2 réservoirs (vol. 1 000 chacun) 2 000 1 000 Double du réel
10 réservoirs (total 470 000) 52 222 L 47 000 L +11,1 % de surestimation
Les 3 assertions demandées par l'énoncé 
# 1. Cas simple : 2 réservoirs de même volume → moyenne = ce volume
r2 = [{"volume": 1000, "capacite": 5000}, {"volume": 1000, "capacite": 5000}]
assert volume_moyen(r2) == 1000   # ÉCHOUE avec le bug (renvoie 2000)

# 2. Moyenne ≤ volume maximal présent dans la liste
max_vol = max(r["volume"] for r in reservoirs)
assert volume_moyen(reservoirs) <= max_vol   # ÉCHOUE si moyennage incorrect

# 3. Moyenne ≤ capacité maximale
max_cap = max(r["capacite"] for r in reservoirs)
assert volume_moyen(reservoirs) <= max_cap   # passe dans ce cas précis

Correction — Question 3

La correction en un mot : remplacer (len(reservoirs)-1) par len(reservoirs). La formule de la moyenne est \(\bar{x} = \frac{\sum x_i}{n}\), pas \(\frac{\sum x_i}{n-1}\) (ce dernier est le dénominateur de la variance corrigée, sans rapport ici).

Stratégie de gestion pour le district Tepua

Le district Tepua est vulnérable à cause de Motuavai (11,1 % — en pénurie) et de Matavai (35,7 %). Voici les leviers d'action possibles :

  3. Alertes automatiques : utiliser est_en_penurie et districts_vulnerables dans un tableau de bord qui déclenche des alertes dès qu'un seuil critique est atteint.
  4. Priorisation des captages : orienter les captages naturels vers les réservoirs de Tepua pendant la prochaine saison des pluies.

Pièges fréquents & stratégie de performance

Piège 1 — Diviser par n-1 au lieu de n pour la moyenne

C'est le bug de ce sujet. La moyenne d'une liste de \(n\) valeurs se calcule en divisant par \(n\), pas par \(n-1\). Le diviseur \(n-1\) est utilisé pour la variance corrigée en statistiques — notion hors-programme NSI. Sur une liste d'un seul élément, \(n-1 = 0\) provoque une ZeroDivisionError.

Piège 2 — Condition stricte vs non stricte

La pénurie est définie « strictement inférieure à 20 % » → taux < 20. Un réservoir à exactement 20,0 % n'est pas en pénurie. Toujours vérifier les cas limites dans les tests : Vaihere est à 21,05 % (pas en pénurie) et Motuavai à 11,11 % (en pénurie).

Piège 3 — Oublier d'initialiser la clé du dictionnaire

Dans volume_par_district, si tu tentes directement volumes[d] += r["volume"] sans initialiser volumes[d] = 0 lors de la première rencontre du district, Python lève une KeyError. La vérification if d not in volumes: volumes[d] = 0 est indispensable.

Piège 4 — Réutiliser les fonctions fournies

Le code fourni contient déjà reservoirs_par_district qui regroupe les réservoirs par district. Pour Q4, il faut réutiliser cette fonction plutôt que de la réécrire. De même, taux_remplissage peut être appelé dans est_en_penurie. Lire toutes les fonctions fournies avant de coder.

# Q1 — est_en_penurie
def est_en_penurie(reservoirs, nom):
    for r in reservoirs:
        if r["nom"] == nom:
            return r["volume"] / r["capacite"] * 100 < 20

# Q2 — volume_par_district
def volume_par_district(reservoirs):
    volumes = {}
    for r in reservoirs:
        d = r["district"]
        if d not in volumes: volumes[d] = 0
        volumes[d] += r["volume"]
    return volumes

# Q3 — volume_moyen corrigé (1 mot changé)
moyenne = somme_totale / len(reservoirs)   # ← supprimer le -1

# Q3 — résultats : bugué=52222 L, correct=47000 L

# Q4 — districts_vulnerables
def districts_vulnerables(reservoirs, seuil_relatif=0.80):
    seuil = volume_moyen(reservoirs) * seuil_relatif
    rpd   = reservoirs_par_district(reservoirs)
    return [d for d, ress in rpd.items() if volume_moyen(ress) < seuil]
